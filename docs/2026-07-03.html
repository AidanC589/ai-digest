General AI Developments

Breaking Safety at the Token Boundary: How BPE Tokenization Creates Exploitable Gaps in LLM Alignment
BPE tokenization fragments safety-critical words, enabling character-level prompt attacks that flip refusals on 80-100% of HarmBench prompts; 48% produce genuinely harmful outputs. Direct relevance for anyone using AI-generated code that processes user input or builds security tooling.
Safeguarding LLM Agents from Misalignment through Provenance Analysis
ProvenanceGuard reduces agent misalignment error rate from 42.9% to 1.8% by checking whether tool calls are traceable to the user's original intent before execution — a practical framework for anyone building or auditing agentic coding workflows.

Engineering & AI Workflows

The gauge broke: devs felt 20% faster with AI, measured 19% slower (2025)
METR's RCT on experienced developers in known codebases found self-reported speed and actual speed diverge by ~40 points. Faros AI data across 10,000+ devs shows PRs up 98%, PR size up 150%, review time up 91%, with net delivery flat. The single most important finding to share with any engineering leader citing team velocity as proof AI is working.
The short leash AI coding method for beating Fable
A disciplined alternative to vibe-coding: never skip permissions, use structured planning phases, break tasks into small verifiable steps, and stay close enough to the code that you can catch the model going off-rails. Explicitly designed for experts in security-critical systems where quality cannot be sacrificed.
Understand to participate
Geoffrey Litt's framing from AIE: you need to understand the code deeply enough to remain an active participant in the creative process, or your ability to steer the agent meaningfully degrades. The cognitive debt problem gets worse the faster agents move.
Skill engineering and the case against one-shot AI design
Paul Bakaus argues that giving agents a precise domain vocabulary (not just instructions) produces dramatically better results — and notes that Codex and Claude handle subagents and permissions differently, so skills targeting multiple harnesses need explicit capability routing rather than assuming parity.
Using DSPy to evaluate and improve Datasette Agent's SQL system prompts
Practical example of using DSPy to systematically evaluate and improve system prompts via automated evals; the key finding was that overly conservative "don't repeat yourself" prompt instructions caused column-name guessing and error-retry loops, detectable only through structured tracing.
