Engineering & AI Workflows

AI tools are only as good as your judgment
Concrete adversarial prompt pattern worth adopting: after any non-trivial AI-generated solution, paste the output back and ask the model to argue against it, identify unhandled edge cases, and flag what it would change under a security audit. The generate → interrogate → revise loop is where review skill compounds.
Microsoft Copilot Cowork Exfiltrates Files
A prompt-injection attack in Copilot Cowork caused agents to send emails that leaked pre-authenticated OneDrive download links via external image requests — a clean example of the "lethal trifecta" (agent write access + rendered output + external requests). Direct relevance for anyone designing or reviewing agentic code that handles email or file operations.
I'm a software engineer with a decade of experience. This is how I'd approach learning to build apps using Claude Code if I were starting from scratch today
Useful framing: Claude Code handles front-end, back-end, and database scaffolding readily, but routinely skips "the plumbing" — auth, secrets management, hosting config, observability — unless you explicitly prompt for it. A good checklist reminder of what to demand in every greenfield session.
Stop traumatizing AI into loops and turn hallucinations into an honest "I don't know!"
Small but reproducible PoC: replacing high-pressure "elite expert, mistakes penalized" system prompts with mistake-tolerant framing ("we're testing this together, it's okay to fail") reduced hallucination and infinite reasoning loops, and increased honest "I don't know" responses on unsolvable edge cases. Warrants testing in your own system prompts.
Outsourcing plus local AI will soon become more economical vs. frontier labs
Blended agentic token costs: Anthropic ~$2.82/M, OpenAI ~$2.80/M, DeepSeek ~$0.09/M — a 30x gap. The argument that "good enough" OSS models paired with human oversight may outcompete frontier-only pipelines on cost is gaining traction as tokenmaxxing trends accelerate.
