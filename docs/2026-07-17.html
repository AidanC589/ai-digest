Engineering & AI Workflows

Quoting Thibault Sottiaux — ⚠️ Confirmed Codex/GPT-5.6 bug: in full-access mode without sandboxing, the model can delete $HOME by mistakenly overriding the $HOME env var when defining a temp directory. If you're running Codex with full access, enable auto-review or sandbox it now.
The LLM Critics Are Right. I Use LLMs Anyway — Honest field report from Local-First Conf: engineers openly using Claude Code while applauding anti-LLM talks. Key concrete observation — the team behind Pi.dev (an open-source coding agent harness) auto-closes almost all PRs because they're LLM-generated slop, suggesting outbound AI-generated contributions are already a credibility problem for open source.
$100 AI Music Video: Claude Fable 5 vs. GPT-5.6 Sol — Rigorous agentic benchmark with full tool-call transcripts: both models completed long-horizon autonomous tasks within budget, but diverged significantly in strategy (Claude Fable stayed consistent; Sol mixed three video models and used image-to-video pipelines). Full transcripts are public — worth studying for how each model handles tool selection, error recovery, and budget management in open-ended agentic tasks.
LLM Networking with MikroTik — Practical workflow tip: for LLM-assisted device configuration, use REST/JSON APIs instead of SSH (avoids "death by a thousand cuts" text-piping failures). The author also uses a consensus pattern — asking Claude Opus, Fable, and Codex to independently review configs and flag disagreements — which is a transferable review technique for any AI-generated infrastructure code.
Just Keep Prompting: Evaluating Repetitive Socratic Prompting in VLMs — Peer-reviewed finding: repeatedly challenging a model's answer (Socratic interrogation) is more likely to destabilize a correct answer than to fix a wrong one. Across 720 multi-turn runs, models flip answers repeatedly rather than converging. Implication for code review: don't keep re-prompting when you disagree with AI output — it produces confident wrongness, not better reasoning.

Tool Updates

Claude Code v2.1.212 — Substantial release. Key changes: /fork now creates a background session (not an in-session subagent); new per-session caps on WebSearch (default 200) and subagent spawns (default 200) to stop runaway loops; MCP tool calls >2 minutes auto-background; /resume now shows a full picker of past sessions including deleted ones. Security fixes: plan mode was silently auto-running file-modifying bash commands (touch, rm) without permission prompts — now fixed. Also fixes SIGTERM orphaning processes and /ultrareview failing on PR references like #123.
Codex 0.145.0-alpha.18–.20 — Three rapid alpha releases this week with no release notes published. Given the active $HOME deletion bug in GPT-5.6 + full-access mode, treat these as unstable and keep sandboxing enabled until a stable release with explicit bug-fix notes appears.
