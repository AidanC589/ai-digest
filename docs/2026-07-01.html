Engineering & AI Workflows

What's new in Claude Sonnet 5
Headline watch-out: Sonnet 5's new tokenizer produces ~30% more tokens than Sonnet 4.6 for the same input, making English text ~1.4x more expensive and Python code ~1.28x more expensive despite unchanged list pricing. Also: temperature, top_p, and top_k sampling parameters are no longer supported, and adaptive thinking is on by default — you must explicitly pass "thinking": {"type": "disabled"} to turn it off.
Sonnet 5 today, and Fable 5 tomorrow
Beyond the model itself, Anthropic shipped significant platform updates alongside the launch: Managed Agents now supports streaming session deltas, per-session overrides, webhook events, reverse pagination, credential injection scoping, and an observability tab with token/tool metrics — making this as much an integration release as a model release.
Have your agent record video demos of its work with shot-scraper video
Practical pattern for closing the agent feedback loop: use shot-scraper video with a YAML storyboard to have agents produce Playwright-recorded video demos of their own work. The storyboard itself was generated by GPT-5.5 via Codex Desktop from a single prompt — a useful template for making agent outputs verifiable without manual QA.
A Single Rewrite Suffices: Empirical Lessons from Production Skill Description Optimization
Production finding directly applicable to prompt engineering: on a 9-skill enterprise agent, a single LLM rewrite of skill descriptions using false-positive and false-negative examples matched the performance of 120-minute manual tuning sessions (79.2% vs 79.4% F1) — reducing effort to 3.8 minutes. Additional iteration, dual editing of confused pairs, and larger training sets each contributed less than 0.5% F1 gain.
Ahmad Osman on why local AI is catching up
Practical caution for teams considering local model routing: running a model locally replicates only a fraction of what Claude Code or similar tools provide — the surrounding infrastructure (context management, tool calling, session handling) is a significant, often underestimated engineering lift.
