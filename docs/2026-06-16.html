The recurring theme across engineering teams right now: tie your architecture to a learning loop, not a specific model. Multiple teams are hardening model-neutral routing after the Fable 5 outage — if you're not abstracting your model calls behind a routing layer, this week is a good time to start.
Claude Fable 5 distilled
Someone distilled 4,659 agentic-coding traces from the brief Fable 5 public window into an open-weights Qwen3.5-35B model (Qwable-v1, AGPL-3.0 on HuggingFace), capturing Fable's tool-use format including str_replace_editor. Interesting as a signal that agentic coding trace data has real distillation value — and that anti-distillation classifiers have limits.

Tool Updates

Claude Code v2.1.178v2.1.178
Notable security improvement: subagent spawns are now evaluated by the classifier *before* launch, closing a gap where a subagent could request blocked actions without review. Also adds Tool(param:value) permission syntax for fine-grained control (e.g. Agent(model:opus) to block Opus subagents), fixes compaction not honoring --fallback-model, and fixes several subagent transcript/backgrounding bugs. Multiple MCP and OAuth fixes too.
Codex 0.140.0
Adds /import to selectively pull setup, project config, and recent chats from Claude Code (direct migration path), /usage views for token activity, permanent session deletion, and managed Amazon Bedrock auth. Also removed experimental realtime voice controls. Performance improvements for large repos and long sessions.
