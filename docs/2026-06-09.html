General AI Developments

Apple reveals new AI architecture built around Google Gemini models
Apple's revamped Apple Intelligence uses co-developed Gemini-derived foundation models running on Private Cloud Compute extended to Google Cloud NVIDIA GPUs; vision LLMs extract screen context without requiring per-app integration, and all PCC binaries will be published for public inspection. Simon Willison's WWDC notes add useful skepticism given 2024's unfulfilled promises, but notes the approach is at least technically feasible today.
FrontierCode: Benchmarking for Code Quality over Slop
Cognition's new benchmark, built with open-source maintainers (40+ hours per task), evaluates whether AI-generated code is actually *mergeable* — scoring on regression safety, cleanliness, scope, and maintainability. Best model scores ~13% on hardest tasks vs. 50%+ on SWE-bench, directly quantifying how much passing tests diverges from production-ready code quality.

Engineering & AI Workflows

FrontierCode: Benchmarking for Code Quality over Slop
The broader meta-lesson from FrontierCode: practitioners are shifting from one-shot prompts toward explicit goal/verification/iteration loops ("state machines not loops"), with Claude Code's own retrospective on auto-mode and OpenAI Codex's outcome-first prompting tips both reflecting this pattern. Human checkpoints remain essential outside easily-verifiable domains.
6 free open source repos that cut my Claude Code token costs by up to 90%
Practical token-reduction stack for Claude Code heavy users: ccusage for per-model token visibility, RTK for compressing bash output (claims 60-90% reduction on command tokens), Graphify for building a local codebase knowledge graph so Claude stops re-reading files, and Karpathy's skills repo for constraining file scope and reducing back-and-forth corrections.
Anthropic changed their privacy policy today and there's a specific clause that every Claude user needs to know about
Effective July 8 2026, Anthropic's new policy allows proactive disclosure of conversation data to law enforcement based on internal "good faith belief" — no court order required, no user notification, no appeals process. Relevant if you're using Claude for security research, red-teaming prompts, or any context-sensitive work; review what's in your sessions before then.
Ask HN: What are tools you have made for yourself since the advent of AI?
Useful snapshot of what engineers are actually shipping with AI assistance: sandboxed agent runners (separate macOS user accounts with sandbox-exec), custom music player plugins, Ollama honeypots for threat intel, and full static blogs. Notably, a sandboxed agent project called sandvault runs agents in isolated macOS user accounts with headless browser support — worth watching for anyone concerned about Claude Code's file system access.

Tool Updates

Claude Code v2.1.169v2.1.169
Large release: new --safe-mode / CLAUDE_CODE_SAFE_MODE flag disables all CLAUDE.md, plugins, hooks, and MCP servers for clean troubleshooting; /cd command changes working directory without breaking prompt cache mid-session; disableBundledSkills setting hides built-in commands from the model. Key fixes: enterprise MCP policies not being enforced on reconnect, a 30-50ms UI stall per turn on macOS with claude.ai credentials, background agents ignoring project-level env settings, and untrusted project settings being able to set OTEL cert paths without trust confirmation (security fix).
Codex v0.138.0v0.138.0
/app command now hands off CLI threads to Codex Desktop on macOS and Windows; local image paths are now exposed to the model for more reliable follow-up edits; large MCP/Ollama streams and long message histories process faster via optimized byte scanning; resume --last now queries state DB first for faster restore on large histories.

Next time Claude Code is behaving unexpectedly due to a CLAUDE.md instruction, a plugin, or an MCP server you suspect is causing issues, run claude --safe-mode (or set CLAUDE_CODE_SAFE_MODE=1) to start a clean session with all customizations disabled.
