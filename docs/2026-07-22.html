Tool Updates

Claude Code v2.1.217v2.1.217
Security fix: background session isolation was not canonicalizing symlinked working directories, which could let sessions escape their workspace folder — update immediately if you use background sessions. Also fixes: memory leak where truncated MCP tool outputs kept full untruncated results in memory, auto-compact never triggering for Opus 4.8 on Bedrock, Windows auto-update failures leaving claude.exe missing, and a CLAUDE.md brace-expansion bug that could OOM-kill the CLI at startup. New: cap on concurrent subagents (default 20, override with CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS), subagents no longer spawn nested subagents by default, and --max-budget-usd now correctly halts background subagents when the cap is reached.
Codex 0.145.0
Major stable release: paginated thread history with search and sub-agent support, /import can now migrate Cursor and Claude Code settings/MCP servers/sessions in one command, experimental Amazon Bedrock login with GPT-5.6 Sol as default Bedrock model, audio inputs, and stabilized multi-agent V2 with configurable sub-agent models and concurrency. Context overhead reduced via concurrent skill discovery and efficient remote compaction.

Based on the Anthropic Claude Code team's direct advice: audit your existing system prompts and CLAUDE.md files for negative instructions ("don't do X, never do Y") and example lists, then delete them. For Fable 5, Opus 4.8, and likely any similarly capable model, these patterns now actively degrade output quality. Replace them with concise positive descriptions of desired behavior and trust the model's defaults. The Claude Code team reduced their own system prompt by 80% using this approach — start with your most instruction-heavy prompt and cut aggressively.
