Engineering & AI Workflows

Fable's judgement
A tip direct from the Claude Code team: rather than prescribing when to write tests or which model to use, tell Claude Code to "use its own judgement" — it will create a persistent memory file routing implementation work to Sonnet/Haiku subagents and keeping review/design in the top model, cutting token burn significantly without losing quality.
No LLM Code in Dependencies
A maintainer spent 100 hours auditing git-annex's full dependency tree for LLM-generated code, finding incoherent 1,489-line commit messages, silent reversions, and probable copyright-adjacent copy-paste. A direct warning: AI-generated code in upstream dependencies is now a real supply-chain quality and legal risk to track.
AI saves about 3% of your hours, and almost none of it reaches the money
A study linking AI adoption surveys of 25,000 workers to actual payroll data found real but small gains (~2.8% of work hours); the key finding for practitioners is that task-level speedups (40% on structured writing, 14–34% in support) only compound into real value when deliberately applied to high-volume repeatable work and the saved time is explicitly reallocated.
Please stop the AI confidence theater
A grounded counterpoint to AI hype: agents that trigger 50% of the time and require hand-fed context aren't replacing workflows yet. The practical takeaway is to stress-test any AI workflow with "if I removed this tomorrow, would my work actually fall apart?" before building processes around it.
