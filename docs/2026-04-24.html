Engineering & AI Workflows

An update on recent Claude Code quality reports — Simon Willison's analysis of Anthropic's postmortem highlights that a bug introduced March 26 caused Claude to clear its thinking context *every turn* after an idle session — not just once on resume. If you regularly leave long-running Claude Code sessions open for hours or days (common in AI-assisted workflows), this explains weeks of "forgetful" behaviour. Worth reading the full postmortem if you build agentic harnesses.
Post-mortem on recent Claude Code quality issues — Official Anthropic confirmation: three separate harness bugs (not model regressions) caused degraded Claude Code quality over ~2 months. All fixed in v2.1.116+. Usage limits have been reset for all subscribers. Key lesson: harness-level bugs can silently degrade AI output quality in ways that look like model regression.
MeshCore development team splits over trademark dispute and AI-generated code — A real-world open-source schism caused partly by an insider using Claude Code extensively without disclosure, then pursuing trademark ownership. Raises practical questions about AI code provenance, contribution transparency, and governance in community projects — relevant as AI-assisted contributions become the norm.
Extract PDF text in your browser with LiteParse for the web — Simon Willison details his Claude Code + Opus 4.7 workflow for porting a Node.js CLI tool to a browser app: starting with a research conversation on mobile, saving notes to a file, then using notes.md as context for Claude Code with an explicit plan.md step before implementation. A clean, reproducible pattern for AI-assisted porting work.
Why does this CLAUDE.md file have so many stars? — A single CLAUDE.md configuration file repo is sitting at 78.5k GitHub stars, suggesting community consensus is forming around certain system prompt patterns. Worth investigating if you haven't reviewed your own CLAUDE.md setup recently.
