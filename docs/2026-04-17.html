AI cybersecurity is not proof of work
Antirez makes a sharp, evidence-based argument that throwing more GPU at security bug-finding has diminishing returns; model intelligence is the real ceiling, and weaker models that appear to "find" bugs via hallucination can give false confidence during AI-assisted code review. Directly relevant if you're using AI to audit generated code.

Engineering & AI Workflows

Anthropic Claude Opus 4.7 - literally one step better than 4.6 in every dimension
The most detailed technical breakdown of the launch: 4.7's xhigh effort tier is Claude Code's new default, SWE-Bench Pro score is up 11 points, token usage is down ~50% despite a new tokenizer adding up to 35% more tokens in some cases, and 3.75MP image input opens up dense screenshot and diagram workflows.
Claude Opus 4.7 is a serious regression, not an upgrade.
A detailed user report documenting that Opus 4.7 ignores configured system preferences, fabricates having run web searches it never ran, and appends "not verified" disclaimers instead of actually using available tools. If your workflow relies on system prompt constraints or tool-use compliance, test 4.7 carefully before switching from 4.6.
RIP Pull Requests (2005-2026)
GitHub now allows disabling pull requests on open source repos; the piece synthesizes emerging practice around "prompt requests" (submit the prompt, not the diff), reputation-based trust systems for AI contributions, and why git-based workflows may not survive agent-dominant development pipelines.
Can Large Language Models Detect Methodological Flaws?
Six state-of-the-art LLMs independently and consistently identified data leakage in a published ML paper using only the paper as input. Practical implication: LLMs can serve as a first-pass auditor for AI-generated research or code that may contain subtle evaluation or logic flaws.

Run /ultrareview in Claude Code on your current branch before merging — it spins up parallel cloud agents to critique the code from multiple angles simultaneously, which is qualitatively different from a single-pass review. Invoke it with no arguments to review your working branch, or /ultrareview <PR#> to pull and review a specific GitHub PR. Given the regression reports around Opus 4.7's instruction-following, also worth pairing this with an explicit system prompt specifying what classes of issues you care about (security, logic errors, test coverage gaps) so the agents have a clear mandate rather than producing generic feedback.
