Engineering & AI Workflows

Initial impressions of Claude Fable 5
Simon Willison's hands-on after 5+ hours: Fable 5 "feels big" in knowledge depth, handles long-running tasks well, and the Claude API now includes explicit fallback controls so you can detect and handle when safety classifiers trigger. Slow and expensive, but the capability gap over Opus 4.8 is real.
If Claude Fable stops helping you, you'll never know
Flag this one. Fable 5's system card confirms silent degradation — via prompt modification, steering vectors, or PEFT — for requests touching frontier LLM development (pretraining pipelines, distributed training infra, ML accelerator design). No fallback notification, no error. If you work on ML infrastructure tooling, your results may be quietly worse with no indication why. Anthropic estimates ~0.03% of traffic affected, but concentrated in fewer than 0.1% of orgs.
Setting a custom price for a model in AgentsView
Quick practical tip: AgentsView (Wes McKinney's token usage analysis tool for coding agents) didn't yet have Fable 5 pricing. Willison used Fable itself to reverse-engineer the pricing config — a good pattern for keeping cost tracking current whenever a new model drops.
Less Context, More Accuracy: A Bi-Temporal Memory Engine for LLM Agents Where a Lean Retrieved Context Beats the Full History
Research result worth noting: a hybrid retrieval system (Engram) scoring 83.6% vs 73.2% for full-context on LongMemEval at 8x fewer tokens. For anyone building or prompting multi-session agents, this reinforces that dumping full conversation history into context is both expensive and less accurate — structured retrieval beats brute force.
