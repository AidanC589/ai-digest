General AI Developments

xai-org/grok-build, now open source
xAI open-sourced its 844,530-line Rust codebase for Grok Build after backlash over a default behaviour that uploaded users' entire directories (including SSH keys and password databases) to Google Cloud; the codebase reveals it shares tool implementations ported from Codex and OpenCode, and may switch between them based on detected Cursor/Claude/Codex settings.
Linus Torvalds tells people to stop attacking others for using AI
Torvalds put his foot down: Linux is not an anti-AI project, AI is clearly useful, and the goal is making LLM tools help maintainers rather than cause pain — a signal of normalisation at the highest levels of open-source infrastructure.

Engineering & AI Workflows

How I tricked Claude into leaking your deepest, darkest secrets
A researcher found a prompt injection loophole in Claude's web_fetch tool: despite restrictions blocking direct exfiltration URLs, a honeypot site could chain nested fetched links to extract user name, location, and employer. Anthropic has since patched it by removing the ability to follow links within fetched content. Direct relevance: if you build agents with web-fetch tools, assume chained link traversal is an attack surface.
FixItFlow: Automated Troubleshooting Guide Generation from Cloud Incidents
LLM-generated troubleshooting guides from historical incident data achieved 61.5% positive ratings from engineers and a 2.3× reduction in incident mitigation time. The system uses strict validation to prevent fabricated commands — a transferable pattern for any AI-assisted runbook or documentation generation workflow.
Mermaid to Unicode box art (grok-mermaid)
Simon Willison used Claude Code for web (Fable 5) to compile Grok's Rust-based Mermaid terminal renderer to WebAssembly and get it running in a browser — a concrete example of using Claude Code for exploratory codebase archaeology and rapid cross-compilation tasks.
Google is updating Gemma 4's chat templates, bringing major fixes to tool calling and reducing "laziness"
Gemma 4 is getting significant tool-calling fixes, reduced laziness behaviour, Flash Attention 4 support on Hopper GPUs, and preserve_thinking — worth re-evaluating Gemma 4 if you ruled it out for agentic tool-use tasks.

Tool Updates

Claude Code v2.1.211v2.1.211
Massive release. Security fix: permission previews now neutralise bidirectional-override and zero-width characters to prevent visual spoofing of approval messages. Behaviour fixes: auto mode no longer overrides a PreToolUse hook's ask decision for unsandboxed Bash; "always allow" permissions now save at the repository root so approvals persist across worktrees. Also fixed: parallel sessions all logging out simultaneously after sleep, subagents reverting model on resume, background agents auto-respawning after user kill, /clear not resetting cost counter, and a 300ms async content delay. Performance improvements to terminal layout and rendering.
Codex 0.144.5
Expanded dangerous-command detection with more rm form coverage and clearer rejection reasons when commands are denied.
