General AI Developments

CEOs admit AI had no impact on employment or productivity
A February NBER study of 6,000 executives found nearly 90% saw no productivity or employment impact from AI over three years, with usage averaging just 1.5 hrs/week — Solow's productivity paradox redux. Individual productivity gains in controlled studies (MIT's ~40%) aren't showing up in macro data yet.

Engineering & AI Workflows

Uber's Anthropic AI push hits a wall
Uber exhausted its AI tooling budget months into 2026 after encouraging engineers to use Claude Code and Cursor via internal usage leaderboards. Claude Code became dominant while Cursor plateaued; Uber is now trialing OpenAI Codex as a cost hedge. 11% of live backend code updates are now AI-written — a real adoption benchmark to compare against.
Claude Token Counter, now with model comparisons
Opus 4.7's updated tokenizer inflates text token counts ~1.46x over 4.6, making it effectively ~40% more expensive despite identical per-token pricing. High-res images get hit harder (3x), but small images and PDFs are much closer to parity. Worth running your own prompts through the comparison tool before committing to 4.7 for cost-sensitive workloads.
Is anyone getting real coding work done with Qwen3.6-35B-A3B-UD-Q4_K_M on a 32GB Mac?
Practical field report: Qwen3.6-35B-A3B on 32GB RAM hits a hard wall at 32K context, losing task state on the second compaction pass. The model's own card recommends 128K minimum for complex tasks — a useful constraint to know if you're evaluating local models as a Claude Code cost-reduction strategy.
YSK: If you use Claude on your company's Enterprise plan, your employer can access every message you've ever sent, including "incognito" chats
Claude's Compliance API lets enterprise admins programmatically pull full chat history, uploaded files, and activity logs — including "incognito" sessions. If you're putting sensitive code, credentials, or architecture details into Claude at work, assume it's logged and accessible to your org.
Headless everything for personal AI
Salesforce and others are shifting to API/MCP/CLI-first surfaces explicitly for AI agent access. The practical implication: evaluating tools for your AI workflow should now include "does it have an API or MCP endpoint?" as a first-class criterion, not an afterthought.
