Engineering & AI Workflows

**Using Claude Code: The Unreasonable Effectiveness of HTML**
A member of the Claude Code team at Anthropic argues for requesting HTML output instead of Markdown, enabling SVGs, interactive widgets, inline annotations, and severity-coded diffs in PR reviews. The example prompt — asking Claude to produce an HTML artifact reviewing a PR with inline margin annotations and color-coded findings — is directly usable today and is a significant upgrade over plain Markdown responses.
**Show HN: Git for AI Agents**
Open-source tool built specifically for Claude Code that adds version control semantics to agent sessions: rewind after /compact, bisect to find when/why something changed, and answer "why did you do that?" Currently 97 points on HN with active discussion — fills a real gap if you've ever lost agent context.
**How do you usually get around when starting big projects in Claude Code?**
Practical community thread on structuring large Claude Code sessions: the consensus leans toward building functional specs as .md files before touching Claude, rather than prompt-dumping. Worth reading if you're scaling from small scripts to multi-component systems.

Next time you ask Claude to review a PR or explain a complex piece of code, replace your usual prompt with one that explicitly requests HTML output: *"Review this PR and produce a styled HTML page with the diff rendered inline, margin annotations for each finding, and findings color-coded by severity (red/yellow/green). Include an in-page nav if there are more than five findings."* Open the response in a browser via a quick pbpaste > review.html && open review.html (or the Claude Code artifact viewer). You'll get SVG diagrams, collapsible sections, and scannable severity colours instead of a wall of Markdown — materially faster to review than text output, especially for PRs touching unfamiliar logic.
