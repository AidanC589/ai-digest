Engineering & AI Workflows

Claude Tag: Multiplayer, Proactive, Persistent Agents in Slack
Claude Tag turns Claude into an async Slack teammate: it can be tagged into threads, wait days on blocking dependencies via git webhooks, monitor A/B tests, tag in human coworkers who own related code, and operate in ambient mode (proactively syncing across channels). Anthropic's own team reports it now merges 65% of product PRs — treat this as a serious workflow shift, not a chatbot wrapper.
What I'm Finding About LLM Code Style and Token Costs
Concrete finding: AI-generated code defaults to verbose patterns (4-space indents, legacy polyfills for platform-native APIs) that inflate output tokens 3–5x vs. input cost. Steering Claude toward modern runtime APIs (Web platform, Deno-style) and compact formatting can meaningfully cut API spend — worth adding explicit style constraints to your system prompts.
simonw/browser-compat-db
Simon Willison used Claude Code (Opus 4.8) to generate a conversion script and Codex Desktop (GPT-5.5) to build a GitHub Actions workflow, then force-pushed the output to an orphan branch for CDN hosting — a clean example of routing different tasks to different agents based on their strengths within a single pipeline.
Quoting Tom MacWright
A sharp signal on the identity problem with fully AI-generated output: job applications now chain LLM-written resumes → LLM portfolio sites → LLM GitHub commits, rendering candidates completely opaque. Same dynamic applies to AI-generated code PRs — reviewers increasingly can't infer intent, context, or judgment from the artifact alone. Worth thinking about how you leave a human signal in AI-assisted work.
Perfect Detection, Failed Control: The Geometry of Knowing vs. Steering in Language Models
Research finding with practical implications for AI code review: a model can detect hallucinations with near-perfect accuracy (AUC=1.0 from layer 5) yet the detection direction sits ~83° away from the direction that produces refusal. Knowing a model "knows" something is wrong doesn't mean it will act on it — relevant to trust calibration when using AI to review AI-generated code.
