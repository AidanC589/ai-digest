Engineering & AI Workflows

Changes in the system prompt between Claude Opus 4.6 and 4.7
Key behavioral shifts in Opus 4.7 you should know: Claude is now instructed to act first and ask fewer clarifying questions (only block when info is genuinely missing), will call tool_search before claiming it lacks a capability, produces ~35% fewer output tokens, and gets more explicit safety guardrails around child safety that persist for the rest of a conversation once triggered.
Claude system prompts as a git timeline
Simon Willison used Claude Code to convert Anthropic's published system prompt history into a git repo with timestamped commits per model — a practical workflow for anyone who wants to git diff prompt changes rather than manually parsing markdown.
The Two Sides of OpenClaw
Opus 4.7 hit #1 on Code Arena (+37 over 4.6) and ranks first on agentic benchmarks, but had a noisy first 24 hours with context failures and regressions that were patched quickly; also: Codex Computer Use is drawing serious practitioner attention for driving Slack, browsers, and legacy desktop apps in enterprise settings.
Qwen3.6-35B-A3B solved coding problems Qwen3.5-27B couldn't
Users running Qwen3.6-35B-A3B locally via OpenCode report it handles multi-agent code review, security analysis, and complex bug-fixing that smaller models couldn't crack — with the added benefit of keeping codebases off third-party servers entirely.
I'm running qwen3.6-35b-a3b with 8 bit quant and 64k context thru OpenCode on my mbp m5 max 128gb
On M5 Max MacBook Pro with 128GB RAM, Qwen3.6 at 8-bit quant is being used as a daily driver for long agentic research and multi-tool coding sessions, with users specifically citing privacy as a reason to switch from cloud providers.

With Opus 4.7's new system-level instruction to act rather than ask clarifying questions, update your prompts to lean into this: instead of front-loading all your constraints hoping Claude will ask follow-ups, write a short intent statement followed by explicit constraints as a bulleted list, then end with "proceed without asking — make reasonable assumptions and note them." This matches how Opus 4.7 is now tuned to behave, so you'll get a complete first attempt with documented assumptions rather than an interrogation — which is far easier to steer in review.
