The zkVM audit article makes a concrete, transferable point: when asking Claude Code to audit a complex multi-module codebase, don't prompt for a bug list per module. Instead, instruct each subagent to output a *structured assumptions document* — what invariants the module relies on, what it delegates to callers, and what it silently expects from its environment. Then have a top-level agent synthesise those documents before looking for cross-module violations. Try this prompt as a subagent instruction: *"Do not report bugs. Instead, document: (1) every precondition this module assumes but does not enforce, (2) every invariant it relies on from its callers, (3) every implicit contract with its dependencies. Be specific enough that a security reviewer could spot a violation in the calling code."*
