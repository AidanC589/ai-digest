Engineering & AI Workflows

AI coding at home without going broke
The practical case for a hybrid spend model: use frontier subscriptions (Claude/ChatGPT) for spec writing and hard reasoning, then route mechanical fill-in work to cheaper open-source models via OpenRouter API. The author claims this can match a 20-engineer team's output for ~$1k/month using spec-driven development.
Codebase getting larger - Qwen3.6-27B starting to compound issues
Real-world account of vibe-coding a codebase into trouble: as context grows past ~50% of 128K, error rates compound. The fix that worked: scoped conversations per function/line range rather than loading the full project, plus removing KV quantization. A useful reminder that AI-generated codebases need structural review checkpoints, not just end-to-end generation.
Mapping SQLite result columns back to their source `table.column`
Simon Willison assigned Claude Code (Opus 4.8) a genuinely hard reverse-engineering problem — mapping SQL query result columns back to source tables including CTEs — and got multiple working solutions using apsw, ctypes, and EXPLAIN. A concrete demonstration of Claude Code tackling non-trivial architecture research, not just boilerplate.
Publishing WASM wheels to PyPI for use with Pyodide
Willison used Codex + GPT-5.5 to package a C++ Luau WASM project and publish it to PyPI via GitHub Actions. The notable workflow detail: iterative AI-assisted packaging of native code for a new platform (pyemscripten) that has minimal existing examples to train on — a stress test of how well these tools handle novel, documentation-sparse tasks.
