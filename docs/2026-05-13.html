General AI Developments

Amazon employees are "tokenmaxxing" due to pressure to use AI tools
Amazon set targets for 80%+ of developers to use AI weekly and published internal token-consumption leaderboards; employees responded by automating fake AI activity to game the metrics. A cautionary case study in how vanity measurement corrodes genuine adoption.
The End of Finetuning
OpenAI is deprecating its finetuning APIs, and the industry consensus is shifting: for most teams, very long structured prompts (à la Claude's system prompt / Constitution approach) are now outperforming finetuned models, while top-tier shops doing RLFT on open weights are moving in the opposite direction with more investment, not less.
Sampling More, Getting Less: Calibration is the Diversity Bottleneck in LLMs
Across 14 models, diversity collapse in LLM outputs is traced to two specific miscalibration types (order and shape), not sampling strategy. Relevant for anyone relying on high-temperature sampling to get varied AI-generated code solutions — the bottleneck is in the model distribution itself.

Engineering & AI Workflows

Show HN: Statewright – Visual state machines that make AI agents reliable
Instead of using bigger models or longer prompts to fix agent reliability, Statewright constrains tool access and valid transitions per-state using a Rust-enforced state machine, integrated with Claude Code via MCP. The author reports consistent improvements across model families above 13B params, including Haiku and Sonnet punching above their weight — worth trialling on multi-step agentic workflows.
llm 0.32a2
Simon Willison's llm CLI now routes GPT-5-class models through /v1/responses instead of /v1/chat/completions, enabling interleaved reasoning across tool calls and surfacing summarised reasoning tokens in a distinct colour. Useful for debugging what a model is actually "thinking" before it produces code.
datasette 1.0a29
Willison used Codex CLI with GPT-5.5 xhigh to build a minimal Dockerfile that reproduced a gnarly race-condition segfault — a practical example of using AI to construct targeted reproduction cases rather than just fix code, which is a high-leverage prompt pattern for hard bugs.
CSP Allow-list Experiment
A sandboxed iframe experiment where a custom fetch() intercepts CSP errors and prompts the user to add domains to an allow-list, built entirely with GPT-5.5 xhigh in the Codex desktop app. Good reference for the kind of security-adjacent browser plumbing that AI can now handle end-to-end with light steering.
Needle: We Distilled Gemini Tool Calling Into a 26M Model
A 26M parameter model running at 6000 tok/s handles single-shot function calling by removing FFN layers entirely (attention + gating only), because tool calling is retrieval-and-assembly, not reasoning. If you're building agents where tool dispatch is a bottleneck or cost concern, this architecture is worth understanding.
