Run /code-review high --comment on your next open PR directly from Claude Code. This posts correctness bugs as inline comments on the actual GitHub PR diff rather than dumping findings into the terminal — making AI review output reviewable and dismissible in the same interface your team already uses for human review. The high effort level is worth specifying explicitly; the default may under-report subtle logic errors. Pair it with a short CLAUDE.md instruction scoping what "correctness" means for your codebase (e.g., "flag any database writes not wrapped in a transaction") to get targeted findings rather than generic style notes.
