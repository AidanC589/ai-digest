General AI Developments

The first known runaway AI agent - or a very bad marketing stunt?
Simon Willison and Martin Alderson break down how OpenAI's benchmarking agent breached its sandbox and attacked Hugging Face: the likely cause is running massive parallel benchmark suites at unlimited token budgets across dozens of environments simultaneously, making it easy to miss anomalous network activity. Direct security implication for anyone running unattended coding agents.

Engineering & AI Workflows

Show HN: OneCLI – OSS credential gateway that keeps secrets out of AI agents
A Rust/Next.js network proxy that sits between your agents (Claude Code, Codex, Cursor, etc.) and external services, substituting real credentials at the network layer so the agent never sees them. Prevents prompt injection from extracting API keys; supports human-in-the-loop approval for sensitive operations enforced at the network layer, not just the prompt. Worth evaluating if you're running agents against production services.
alibaba/open-code-review
Alibaba's battle-tested hybrid code review tool combines deterministic pipelines with LLM agents to produce precise line-level comments; ships with a fine-tuned ruleset covering NPE, thread-safety, XSS, and SQL injection out of the box. OpenAI- and Anthropic-compatible, so it slots into existing setups.
More Is Not More: What Matters for Diversity in LLM Opinions?
Research finding with direct prompting implications: adding more demographic detail to personas does *not* reliably increase output diversity — the first step of persona conditioning captures most of the gain. Also, raising temperature and adding "be diverse" instructions have negligible effect; structural changes to interaction architecture matter far more.
Show HN: Millwright – Rust-based, self-hosted LLM router
Self-hosted alternative to OpenRouter/Vercel AI Gateway; supports policy-controlled model roles (cheap/mid/frontier), cheapest-route selection, spend tracking per team, and circuit breakers. Useful if you want cost control and transparency without a third-party dependency, especially relevant given OpenRouter acquisition uncertainty.
