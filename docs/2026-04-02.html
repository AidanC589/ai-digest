Engineering & AI Workflows

  • [[AINews] The Claude Code Source Leak](https://www.latent.space/p/ainews-the-claude-code-source-leak) — The accidental leak of Claude Code's orchestration layer (not model weights) exposed concrete architectural decisions worth studying: a 3-layer memory system with auto-compaction ("autoDream"), fork-join subagents that reuse KV cache making parallelism nearly free, fewer than 20 default tools, and aggressive prompt cache reuse — all signal-rich patterns for anyone designing AI coding workflows or agent harnesses.
  • [[AINews] The Claude Code Source Leak](https://www.latent.space/p/ainews-the-claude-code-source-leak) — The leaked code confirms Claude Code puts repo state (recent commits, git branch info) directly in context and uses file read deduplication and structured session memory — practical prompt engineering patterns you can replicate in your own workflows today without waiting for Anthropic to document them.
  • [[AINews] The Claude Code Source Leak](https://www.latent.space/p/ainews-the-claude-code-source-leak) — Claude Code's 5-level permission system and two distinct Plan modes are now visible; understanding the permission hierarchy matters directly for anyone configuring Claude Code in team or CI environments where trust boundaries need to be explicit.
