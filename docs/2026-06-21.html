When I reject AI code even if it works
A tight, evidence-based list of rejection criteria: diff bigger than the problem, abstractions added before they're proven necessary, code that passes CI but degrades system reasoning. The key insight: running the agent *twice* — once to build context, once to actually solve — produces far better output than trusting the first session.
Temporary Cloudflare accounts for AI agents
Cloudflare now lets agents run wrangler deploy --temporary to spin up a live Worker for 60 minutes with no auth friction; the claim URL is handed back to the human. This closes the write→deploy→verify loop for background agents and is a meaningful workflow unblock for agentic coding sessions that need real deployment targets to curl and verify.
