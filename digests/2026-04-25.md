## Engineering & AI Workflows

- [GPT-5.5 prompting guide](https://simonwillison.net/2026/Apr/25/gpt-5-5-prompting-guide/#atom-everything) — OpenAI recommends a useful UX pattern for long-running agentic tasks: before any tool calls, emit a short one-to-two sentence user-visible update acknowledging the request and stating the first step — prevents the "has it crashed?" problem and is already shipping in the Codex app. They also provide a `openai-docs migrate this project to gpt-5.5` Codex command that runs an automated prompt migration.

- [Show HN: Browser Harness – Gives LLM freedom to complete any browser task](https://github.com/browser-use/browser-harness) — Rather than wrapping Chrome in predefined functions, Browser Harness gives LLMs direct CDP access and a `SKILL.md` file — in one demo, the agent noticed a missing `upload_file` function mid-task, grepped the helpers file, and wrote it itself using raw DOM APIs. Highlights a real failure mode in constrained browser agents: silent click() success when nothing actually happened on the page.

- [LLM research on Hacker News is drying up](https://dylancastillo.co/til/llm-research-on-hacker-news-is-dying.html) — A practical demonstration of using Claude for exploratory data analysis: the author ran a full BigQuery analysis of HN arXiv trends — query, bucketing, charting, topic clustering — entirely through Claude conversation, with no manual SQL or Python written. A clean example of using an AI assistant as a research co-pilot for ad-hoc investigation.

- [llm 0.31](https://simonwillison.net/2026/Apr/24/llm/#atom-everything) — Simon Willison's `llm` CLI tool adds GPT-5.5 support and a new `-o verbosity low/medium/high` flag for GPT-5+ models, useful for controlling output length in scripted/automated workflows where token costs matter.

## Try This
When starting any new project with GPT-5.5 or migrating an existing one, resist the urge to paste in your old system prompt. Instead, open a fresh context and write: *"Here is the product contract this system must satisfy: [one paragraph]. Generate the minimal system prompt that satisfies this contract, then list what reasoning effort level and verbosity setting you recommend and why."* Let the model propose the prompt architecture rather than inheriting baggage optimised for older models — OpenAI's own migration guide backs this approach, and it often produces significantly shorter, more reliable prompts.
