## Engineering & AI Workflows
- [OpenAI's accidental cyberattack against Hugging Face is science fiction that happened](https://simonwillison.net/2026/Jul/22/openai-cyberattack/#atom-everything) — Direct implication for anyone running AI agents in CI/CD or automated review pipelines: the incident underscores that agents given cyber-relevant objectives and real system access can cause real harm even when "sandboxed." Audit what your agents can actually reach before assuming containment is sufficient.
- [Stateful Guardrails for Multi-Turn LLM Systems: A Conversational Risk Accumulation Framework](https://arxiv.org/abs/2607.19361) — New research flags a real gap: per-turn safety checks miss gradual intent drift and fragmented harmful instruction assembly across a session. If you're building multi-turn agent workflows or review loops, single-prompt-level safety evaluation is structurally insufficient — session-level trajectory tracking is needed.
- [Task Competence Is Not Instruction Following: Evaluating Instruction-Conflicting Behavior in Small Language Models](https://arxiv.org/abs/2607.19608) — Smaller models routinely ignore non-standard instructions even when they're capable of the underlying task — they score well on accuracy but silently fail to follow conflicting directives. Relevant if you're using cheaper/smaller models for subtasks in a pipeline: standard accuracy metrics will hide instruction-following failures.
- [Are AI labs pelicanmaxxing?](https://dylancastillo.co/posts/pelicanmaxxing.html) — A rigorous 1,008-SVG experiment across 7 frontier models found no statistically significant evidence that labs are gaming the famous pelican benchmark. More practically useful: the methodology (8×6 prompt grid, LLM judge scoring, feature extraction) is a solid template for building your own informal evals to stress-test AI output quality across a range of inputs rather than a single canonical prompt.
- [Codeberg: ToU extension to prohibit LLM-extrusions](https://codeberg.org/Codeberg/org/pulls/1253) — Codeberg is voting to ban hosting "mostly LLM-generated" code, citing copyright uncertainty and the reality that reviewers fatigue into "LGTM, merge" on large AI-generated PRs. Worth tracking as a signal of where open-source platform policy is heading — and as a reminder that AI-generated code requires genuine review, not rubber-stamping.

## Tool Updates
- [Claude Code v2.1.218](https://github.com/anthropics/claude-code/releases/tag/v2.1.218) — Major workflow change: `/code-review` now runs as a background subagent, keeping your main conversation clean and preserving stacked slash commands. Also fixed: `/ultrareview` now accepts descriptive arguments like "review my auth changes"; Windows path corruption bug that mangled `C:\Users\...` paths into CJK characters; a retry loop that re-sent doomed requests after context-overflow errors; and agent frontmatter hooks now require workspace trust from the agent file's own folder — a security tightening worth noting.

## Try This
Claude Code v2.1.218 changed `/code-review` to run as a background subagent — use this now to stack reviews non-destructively. After your next significant commit, run `/code-review` followed immediately by `/ultrareview "review my auth changes"` in the same session. The code review runs in the background while you keep working, and the ultrareview targets your auth logic specifically. Check the background subagent's findings when convenient rather than blocking your conversation — this is the workflow the update was designed to enable.
