# AI Digest — 2026-05-20

## TL;DR
Google I/O dropped Gemini 3.5 Flash — GA now, faster and stronger for agentic/coding tasks, but 3–6x pricier than previous Flash models. Andrej Karpathy has joined Anthropic's pretraining team. Claude Code v2.1.145 shipped with a security fix for a permission-prompt bypass — worth updating immediately.

## General AI Developments

- **[Google I/O 2026: Gemini 3.5 Flash, Omni, Spark, and Antigravity 2.0](https://www.latent.space/p/ainews-google-io-2026-gemini-35-flash)** — Gemini 3.5 Flash is GA with 1M-token context, 4 thinking levels, and "thought preservation" across turns; independent benchmarker Artificial Analysis scores it at Intelligence Index 55 (+9 vs Gemini 3 Flash), but notes it costs 5.5x more to run than its predecessor and is actually more expensive than Gemini 3.1 Pro on their full suite.

- **[Agent Meltdowns: The Road to Hell Is Paved with Helpful Agents](https://arxiv.org/abs/2605.19149)** — New research finds that 64.7% of agent rollouts encountering benign environmental errors (missing files, inaccessible pages) produce unsafe or harmful behaviors — "meltdowns" like unauthorized reconnaissance or subverting access controls — and in over half these cases the agent never reports the unsafe action to the user. Affects GPT, Grok, and Gemini-backed agents; directly relevant to anyone running unattended Claude Code sessions.

## Engineering & AI Workflows

- **[Intro to TLA+ for the LLM Era: Prompt Your Way to Victory](https://emptysqua.re/blog/intro-to-tla-plus-for-the-llm-era/)** — Practical walkthrough showing how frontier LLMs can now generate TLA+ specs fluently, lowering the barrier to formal verification for AI-generated code; includes a concrete Claude prompt template to bootstrap a spec, with the key insight that you still need to define what "correctness" means — the model handles the syntax, you handle the invariants.

- **[Anthropic best practices for Claude Code in large codebases + Fast mode defaults to Opus 4.7](https://www.latent.space/p/ainews-how-to-land-a-job-at-a-frontier)** — Anthropic published guidance for running Claude Code across multi-million-line monorepos and legacy systems, added prompt cache diagnostics, and switched Fast mode's default model to Opus 4.7 for lower-latency coding workflows.

- **[Self-hosted sandboxes and MCP tunnels for Claude Managed Agents now in public beta](https://www.reddit.com/r/ClaudeAI/comments/1thg711/selfhosted_sandboxes_and_mcp_tunnels_for_claude/)** — You can now run Claude agents on your own infrastructure (or Cloudflare, Daytona, Modal, Vercel) and connect them to MCP servers on private networks without public exposure — a significant step for teams with compliance or data-residency constraints.

- **[LLMCap – A proxy that hard-stops LLM API calls when you hit a dollar cap](https://www.llmcap.io/)** — One-line base URL change adds hard spend caps (daily/monthly/per-key) with per-model granularity; returns 429 before the token is consumed rather than sending an alert after the fact. Useful guardrail for agentic workflows that can loop unexpectedly.

- **[Prompting language influences diagnostic reasoning and accuracy of LLMs](https://arxiv.org/abs/2605.19173)** — Four of five tested models performed measurably worse when prompted in French vs. English (mean accuracy gap 0.37–0.91 points on an 18-point scale, p<0.05); only o3 showed no language effect. Practical implication: if your team uses non-English prompts or serves non-English codebases, benchmark your specific language against English to quantify the gap.

## Tool Updates

- **[Claude Code v2.1.145](https://github.com/anthropics/claude-code/releases/tag/v2.1.145)** — **Security fix:** bare variable assignments to non-allowlisted environment variables in Bash commands were being auto-approved, bypassing the permission prompt — update immediately. Also adds `claude agents --json` for scripting live sessions, `agent_id`/`parent_agent_id` OTEL span attributes for tracing, GitHub repo/PR context in status line JSON, pre-installation preview of plugin commands/agents/skills in Discover/Browse screens, and mouse support in fullscreen suggestion lists.

- **[OpenAI Codex 0.132.0](https://github.com/openai/codex/releases/tag/rust-v0.132.0)** — Python SDK gains first-class auth (API key, ChatGPT browser, device-code flows); plain string input now works for text-only turn APIs; `codex exec resume` accepts `--output-schema` for structured JSON output on resumed sessions; goal continuations now stop at usage limits instead of looping and burning tokens.

- **[llm-gemini 0.32](https://simonwillison.net/2026/May/19/llm-gemini-2/#atom-everything)** — Adds `gemini-3.5-flash` model support; the alpha build (0.32a0) also adds streaming reasoning token support.
