## General AI Developments
- [Building reliable agentic AI systems](https://martinfowler.com/articles/reliable-llm-bayer.html) — Bayer AG published a detailed case study on PRINCE, their production agentic RAG system, introducing the useful framing of "context engineering" (what info reaches each agent) vs "harness engineering" (orchestration, retries, observability, human-in-the-loop) — concrete architecture vocabulary worth adopting when designing or prompting multi-agent systems.

## Engineering & AI Workflows
- [When I reject AI code even if it works](https://vinibrasil.com/when-i-reject-ai-code-even-if-it-works/) — A tight, evidence-based list of rejection criteria: diff bigger than the problem, abstractions added before they're proven necessary, code that passes CI but degrades system reasoning. The key insight: running the agent *twice* — once to build context, once to actually solve — produces far better output than trusting the first session.
- [Temporary Cloudflare accounts for AI agents](https://blog.cloudflare.com/temporary-accounts/) — Cloudflare now lets agents run `wrangler deploy --temporary` to spin up a live Worker for 60 minutes with no auth friction; the claim URL is handed back to the human. This closes the write→deploy→verify loop for background agents and is a meaningful workflow unblock for agentic coding sessions that need real deployment targets to curl and verify.
- [Building reliable agentic AI systems](https://martinfowler.com/articles/reliable-llm-bayer.html) — On the engineering side specifically: PRINCE uses reflection loops, named entity recognition for data quality, and explicit state persistence between agents — patterns directly applicable when reviewing or prompting multi-step coding agents to improve reliability.
