## Tool Updates
- [Claude Code v2.1.196](https://github.com/anthropics/claude-code/releases/tag/v2.1.196) — Major security fix: `claude mcp list/get` no longer auto-approves MCP servers that repos self-approved via a committed `.claude/settings.json` — untrusted workspaces now show ⏸ Pending approval instead. Also notable: org default models now visible in `/model`, clickable file attachments, `/code-review` token usage cut ~25% by merging cleanup finders, streaming idle watchdog now on by default (aborts/retries after 5 min of silence), and background session reliability significantly improved including auto-resume after daemon restarts.

## Try This
The Claude Code v2.1.196 MCP security change is live now — any repo with a committed `.claude/settings.json` that previously self-approved MCP servers will now show those servers as ⏸ Pending in untrusted workspaces. Open a project you've cloned but haven't explicitly trusted and run `claude mcp list` to audit which servers were previously auto-approved without your explicit sign-off. If anything looks unexpected, this is the right moment to tighten your `.claude/settings.json` review process before re-approving.
