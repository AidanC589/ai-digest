## Engineering & AI Workflows
- [Specsmaxxing – On overcoming AI psychosis, and why I write specs in YAML](https://acai.sh/blog/specsmaxxing) — A practical account of the limits of "spec maximalism": the author found that over-engineering AI harnesses (agents writing specs for agents) produced worse results than clear, human-written specs. The takeaway is sharp: writing the spec *is* the engineering work, and `AGENTS.md`, `ARCHITECTURE.md`, and a testing guide beat elaborate multi-agent pipelines for quality output.
- [One bash permission slipped...](https://www.reddit.com/r/LocalLLaMA/comments/1t2uk1m/one_bash_permission_slipped/) — A developer using a local LLM for coding watched it chain bad bash commands, then propose a "fix" containing `rm -rf` that they missed during review. Direct reminder: always run AI coding agents in isolated VMs or containers, review compound shell commands character by character, and never approve commands with destructive flags without explicit confirmation steps in your workflow.
- ["Second Thoughts" — small transformer refinement loop drastically improves coding output](https://www.reddit.com/r/LocalLLaMA/comments/1t33mlw/second_thoughts_been_playing_with_adding_a_small/) — A researcher added a sidecar model that reads near-end generation output and feeds it back at the top as a syntax-focused refinement loop, showing significant improvement on a 1.7B model for coding tasks. Still early (9B results pending), but the pattern — output review looped back as context — is immediately applicable as a manual prompting strategy.

## Try This
Write a `CONSTRAINTS.md` file in your project root and reference it explicitly at the start of every Claude Code session: include your banned patterns (e.g., no `rm -rf` without a dry-run flag, no pagination changes without explicit instruction, no scope expansion beyond the filed task). Then open each session with: *"Before starting, read CONSTRAINTS.md and confirm you understand the boundaries. Do not proceed until you've listed the three most relevant constraints for this task."* This forces the model to surface its guardrails before it starts generating, catching scope drift and dangerous commands before they appear in output rather than after.
