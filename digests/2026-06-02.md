## Tool Updates
- [Claude Code v2.1.160](https://github.com/anthropics/claude-code/releases/tag/v2.1.160) — Security-focused release: Claude Code now prompts before writing to shell startup files (`.zshenv`, `.zlogin`, `.bash_login`) and build-tool configs that grant code execution (`.npmrc`, `.yarnrc*`, `.bazelrc`, `.pre-commit-config.yaml`, `.devcontainer/`). The dynamic-workflow trigger keyword is renamed from `workflow` to `ultracode` (violet-highlighted in input). Also fixed: WSL clipboard via PowerShell interop, background session history loss on resume, CJK IME positioning, and Windows input freezing under heavy CPU load.
- [Codex v0.136.0](https://github.com/openai/codex/releases/tag/rust-v0.136.0) — Adds session archiving (`/archive` command), clickable web links in TUI markdown, Windows sandbox alpha provisioning, and remote execution via `CODEX_API_KEY`. Security hardening prevents `/diff` from running repo-provided Git hooks and rejects browser-origin exec-server WebSocket handshakes. Bedrock auth now falls back to `AWS_REGION`/`AWS_DEFAULT_REGION`.

## Try This
If you're using Claude Code's `ultracode` workflows (formerly triggered by the word `workflow` — now renamed as of v2.1.160), update any saved prompts or scripts that relied on the old keyword. More importantly, test the new security prompts by intentionally asking Claude Code to modify a `.npmrc` or `.pre-commit-config.yaml` — understand exactly when and how it now gates on your approval before touching files that can execute arbitrary code, so you can calibrate your `acceptEdits` settings for your actual risk tolerance rather than discovering the behaviour mid-session.
