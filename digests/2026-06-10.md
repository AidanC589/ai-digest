## TL;DR
Anthropic released Claude Fable 5 — a Mythos-class model now generally available, with exceptional coding performance but two controversies: silent degradation for frontier LLM research tasks, and mandatory 30-day data retention. Update Claude Code to v2.1.170 to get access now.

Direct implication: "the AI said so" is not a legal defense for anyone deploying AI-generated content at scale.

The two asterisks are mandatory 30-day traffic retention and silent RSI suppression for ML infrastructure tasks.

## Engineering & AI Workflows
- [Initial impressions of Claude Fable 5](https://simonwillison.net/2026/Jun/9/claude-fable-5/#atom-everything) — Simon Willison's hands-on after 5+ hours: Fable 5 "feels big" in knowledge depth, handles long-running tasks well, and the Claude API now includes explicit fallback controls so you can detect and handle when safety classifiers trigger. Slow and expensive, but the capability gap over Opus 4.8 is real.
- [If Claude Fable stops helping you, you'll never know](https://simonwillison.net/2026/Jun/10/if-claude-fable-stops-helping-you/#atom-everything) — **Flag this one.** Fable 5's system card confirms silent degradation — via prompt modification, steering vectors, or PEFT — for requests touching frontier LLM development (pretraining pipelines, distributed training infra, ML accelerator design). No fallback notification, no error. If you work on ML infrastructure tooling, your results may be quietly worse with no indication why. Anthropic estimates ~0.03% of traffic affected, but concentrated in fewer than 0.1% of orgs.
- [Setting a custom price for a model in AgentsView](https://simonwillison.net/2026/Jun/9/agentsview-custom-model-price/#atom-everything) — Quick practical tip: AgentsView (Wes McKinney's token usage analysis tool for coding agents) didn't yet have Fable 5 pricing. Willison used Fable itself to reverse-engineer the pricing config — a good pattern for keeping cost tracking current whenever a new model drops.
- [Less Context, More Accuracy: A Bi-Temporal Memory Engine for LLM Agents Where a Lean Retrieved Context Beats the Full History](https://arxiv.org/abs/2606.09900) — Research result worth noting: a hybrid retrieval system (Engram) scoring 83.6% vs 73.2% for full-context on LongMemEval at 8x fewer tokens. For anyone building or prompting multi-session agents, this reinforces that dumping full conversation history into context is both expensive and less accurate — structured retrieval beats brute force.

## Tool Updates
- [v2.1.170 — Claude Code](https://github.com/anthropics/claude-code/releases/tag/v2.1.170) — Adds Claude Fable 5 access. Also fixes a bug where sessions launched from the VS Code integrated terminal (or any shell inheriting Claude Code env vars) weren't saving transcripts, breaking `--resume`. Update immediately if you use VS Code's terminal.
- [Codex 0.139.0](https://github.com/openai/codex/releases/tag/rust-v0.139.0) — Code mode can now call standalone web search directly, including from nested JavaScript tool calls. Tool schemas now preserve `oneOf`/`allOf` for better MCP compatibility. `codex resume --last` and `codex fork --last` now correctly treat trailing arguments as prompts rather than session IDs — a notable UX fix. Sandbox networking proxy enforcement is also more consistent.

## Try This
Open AgentsView and set up custom pricing for Fable 5 today before you run any significant sessions: use `$10` per million input tokens and `$50` per million output tokens. Then, for every multi-step coding task you delegate to Fable, check the treemap after the session to see which sub-tasks are consuming disproportionate tokens — Fable's verbosity and long-context capability means costs can concentrate in unexpected places, and catching that early will help you craft tighter, scope-bounded prompts rather than open-ended delegation.
