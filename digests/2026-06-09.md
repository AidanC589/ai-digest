## Tool Updates

- [Claude Code v2.1.169](https://github.com/anthropics/claude-code/releases/tag/v2.1.169) — Large release: new `--safe-mode` / `CLAUDE_CODE_SAFE_MODE` flag disables all CLAUDE.md, plugins, hooks, and MCP servers for clean troubleshooting; `/cd` command changes working directory without breaking prompt cache mid-session; `disableBundledSkills` setting hides built-in commands from the model. Key fixes: enterprise MCP policies not being enforced on reconnect, a 30-50ms UI stall per turn on macOS with claude.ai credentials, background agents ignoring project-level env settings, and untrusted project settings being able to set OTEL cert paths without trust confirmation (security fix).

- [Codex v0.138.0](https://github.com/openai/codex/releases/tag/rust-v0.138.0) — `/app` command now hands off CLI threads to Codex Desktop on macOS and Windows; local image paths are now exposed to the model for more reliable follow-up edits; large MCP/Ollama streams and long message histories process faster via optimized byte scanning; `resume --last` now queries state DB first for faster restore on large histories.

## Try This

Next time Claude Code is behaving unexpectedly due to a CLAUDE.md instruction, a plugin, or an MCP server you suspect is causing issues, run `claude --safe-mode` (or set `CLAUDE_CODE_SAFE_MODE=1`) to start a clean session with all customizations disabled. This is now the fastest way to isolate whether a problem is in your project configuration or the model itself — previously you'd have to manually comment out sections of CLAUDE.md or disconnect MCP servers one by one. Use it as a first diagnostic step before filing bugs or spending tokens debugging configuration issues.
