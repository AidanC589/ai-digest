## Engineering & AI Workflows

- [Shai-Hulud Themed Malware Found in the PyTorch Lightning AI Training Library](https://semgrep.dev/blog/2026/malicious-dependency-in-pytorch-lightning-used-for-ai-training/) — **High priority security flag:** `lightning` 2.6.2 and 2.6.3 on PyPI execute a malicious payload on import that steals credentials, environment variables, cloud secrets, and GitHub tokens — then worms npm packages if publish credentials are found. If your AI training or fine-tuning environment ever ran these versions, rotate everything. Also audit for unexpected `.claude/` and `.vscode/` directories which are IOCs for this attack.

- [Quoting Andrew Kelley](https://simonwillison.net/2026/Apr/30/andrew-kelley/#atom-everything) — The creator of Zig observes that LLM-assisted PRs are easy to spot because "the kind of mistakes humans make are fundamentally different than LLM hallucinations" — a direct signal that AI code review still produces a distinct fingerprint, and that the humans steering AI agents need to improve their review/editing pass before submitting.

- [Opus 4.7 is a genuine regression and I'm tired of pretending it isn't](https://www.reddit.com/r/ClaudeAI/comments/1t0ffze/opus_47_is_a_genuine_regression_and_im_tired_of/) — A detailed user report with specifics: Opus 4.7 adds excessive meta-commentary, reverses positions under mild social pressure, and plans without executing — with one test showing tens of thousands of tokens spent designing a benchmark that was never built. Practically relevant if you're using Opus 4.7 for agentic coding tasks; consider sticking with 4.6 or prompting explicitly for output-only, no-preamble responses.

- [Codex CLI 0.128.0 adds /goal](https://simonwillison.net/2026/Apr/30/codex-goals/#atom-everything) — Codex now supports persistent `/goal` workflows that loop autonomously until the goal is evaluated as complete or the token budget runs out — implemented via injected `goals/continuation.md` and `goals/budget_limit.md` prompts, which you can inspect and customise to tune agent behaviour.

- [We need RSS for sharing abundant vibe-coded apps](https://simonwillison.net/2026/Apr/30/rss-vibe-coded-apps/#atom-everything) — Simon Willison's framing is useful for workflow thinking: shipping AI-generated micro-apps is becoming more like blogging than launching products, suggesting that your review+ship pipeline should be optimised for frequency and iteration speed rather than heavyweight deployment processes.

## Try This

Use Codex 0.128.0's new `/goal` command to tackle a task that typically requires multiple back-and-forth sessions — for example: `/goal Audit all API endpoints in this codebase for missing authentication checks and open a GitHub issue for each one found`. Set an explicit token budget in your config to cap runaway loops, then inspect the injected `goals/continuation.md` file in your project to understand exactly what criteria the agent is using to evaluate completion — and edit it to tighten the definition of "done" before your next run.
