## Engineering & AI Workflows

- [AI tools are only as good as your judgment](https://theaileverageweekly.com/posts/your-ai-tools-are-only-as-good-as-your-judgment-and-that-s-the-point.html) — Concrete adversarial prompt pattern worth adopting: after any non-trivial AI-generated solution, paste the output back and ask the model to argue against it, identify unhandled edge cases, and flag what it would change under a security audit. The generate → interrogate → revise loop is where review skill compounds.

- [Microsoft Copilot Cowork Exfiltrates Files](https://simonwillison.net/2026/May/26/copilot-cowork-exfiltrates-files/#atom-everything) — A prompt-injection attack in Copilot Cowork caused agents to send emails that leaked pre-authenticated OneDrive download links via external image requests — a clean example of the "lethal trifecta" (agent write access + rendered output + external requests). Direct relevance for anyone designing or reviewing agentic code that handles email or file operations.

- [I'm a software engineer with a decade of experience. A good checklist reminder of what to demand in every greenfield session.

- [Stop traumatizing AI into loops and turn hallucinations into an honest "I don't know!"](https://www.reddit.com/r/LocalLLaMA/comments/1tot20j/stop_traumatizing_ai_into_loops_and_turn/) — Small but reproducible PoC: replacing high-pressure "elite expert, mistakes penalized" system prompts with mistake-tolerant framing ("we're testing this together, it's okay to fail") reduced hallucination and infinite reasoning loops, and increased honest "I don't know" responses on unsolvable edge cases. Warrants testing in your own system prompts.

- [Outsourcing plus local AI will soon become more economical vs. frontier labs](https://www.signalbloom.ai/posts/outsourcing-plus-localai-will-soon-become-more-economical-vs-frontier-labs/) — Blended agentic token costs: Anthropic ~$2.82/M, OpenAI ~$2.80/M, DeepSeek ~$0.09/M — a 30x gap. ## Tool Updates

- [Claude Code v2.1.152](https://github.com/anthropics/claude-code/releases/tag/v2.1.152) — Big release: `/code-review --fix` now auto-applies findings to your working tree (reuse, simplification, efficiency); `/simplify` now calls it internally. New `MessageDisplay` hook lets you transform or suppress assistant output. Skills/slash commands can now restrict available tools via `disallowed-tools` frontmatter. `--fallback-model` now activates silently mid-session rather than failing. Auto mode no longer requires opt-in consent. Vim mode gains reverse history search. Several rendering and focus-mode bug fixes.

- [Codex v0.134.0](https://github.com/openai/codex/releases/tag/rust-v0.134.0) — Adds cross-session conversation history search with case-insensitive content matching and previews. MCP setup gains per-server environment targeting and OAuth for streamable HTTP. Read-only MCP tools now run concurrently when `readOnlyHint` is set. Conversation history now exposed to extension tools. Windows TUI rendering corruption fixed.

## Try This

After Claude Code generates any non-trivial function or module, immediately follow up with: *"Now argue against this solution. What edge cases does it not handle? What assumptions did you make that might not hold in production? What would you change if this were reviewed in a security audit?"* This adversarial loop — drawn from today's workflow piece — forces the model to surface its own blind spots before you review, and trains your eye for what good skeptical review looks like. Run it consistently and you'll start anticipating the gaps before you even ask.
