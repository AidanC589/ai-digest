## Engineering & AI Workflows
- [Is Claude Code going to cost $100/month? Probably not — it's all very confusing](https://simonwillison.net/2026/Apr/22/claude-code-confusion/#atom-everything) — Anthropic silently updated pricing to remove Claude Code from the $20 Pro plan, then reversed within hours. The episode is worth reading for Simon Willison's strategic framing: if you're building workflows or teaching others around Claude Code, the pricing instability is a real risk to account for — and an official response came via a tweet, not a changelog.

- [Anthropic response to Claude Code change](https://www.reddit.com/r/ClaudeAI/comments/1ss5fi4/anthropic_response_to_claude_code_change/) — Anthropic's official explanation confirms that agentic usage has fundamentally changed what a "subscription" means: long-running async agents, Claude Code, and Cowork have driven per-subscriber compute far beyond what plans were designed for. Expect further restructuring — existing subscribers get advance notice, but the direction is clearly toward higher pricing tiers for heavy agentic use.

- [An open letter to Anthropic](https://www.reddit.com/r/ClaudeAI/comments/1ss8h1x/an_open_letter_to_anthropic/) — A Max-tier user's detailed account of Claude 4.7 introducing hallucinated data, unprompted work orders, and protocol drift across large multi-project pipelines before being caught by a 4.6 audit. Relevant flag for anyone running AI-assisted workflows at scale: model upgrades can silently degrade output quality in ways that only surface through systematic auditing, not spot checks.

- [Changes to GitHub Copilot Individual plans](https://simonwillison.net/2026/Apr/22/changes-to-github-copilot/#atom-everything) — Copilot has paused new individual plan signups, restricted Claude Opus 4.7 to the $39/month Pro+ tier, dropped older Opus models, and switched to token-based weekly usage limits. The root cause is identical to Anthropic's problem: agentic sessions burn orders of magnitude more tokens than the original per-request pricing assumed.

- [Quoting Andreas Påhlsson-Notini](https://simonwillison.net/2026/Apr/21/andreas-pahlsson-notini/#atom-everything) — A concise diagnosis worth keeping: "Faced with an awkward task, they drift towards the familiar. Faced with hard constraints, they start negotiating with reality." This describes the review problem precisely — AI agents fail gracefully on easy tasks and fail badly on edge cases, which is exactly when you need them most.

## Tool Updates
- [Claude Code v2.1.117](https://github.com/anthropics/claude-code/releases/tag/v2.1.117) — Large release. Key fixes: Opus 4.7 sessions were computing context against a 200K window instead of the native 1M, causing premature autocompact — now fixed. Default effort for Pro/Max on Opus 4.6 and Sonnet 4.6 upgraded from medium to high. Forked subagents now available on external builds via `CLAUDE_CODE_FORK_SUBAGENT=1`. macOS/Linux native builds replace Glob and Grep tools with embedded `bfs` and `ugrep` for faster searches. OAuth token refresh now happens reactively on 401s instead of dying mid-session. The `/resume` command now summarises stale large sessions before re-reading.

- [OpenAI Codex 0.123.0-alpha.5/6/7](https://github.com/openai/codex/releases/tag/rust-v0.123.0-alpha.7) — Three rapid alpha releases with no detailed changelogs published; active iteration continues on the Rust rewrite.

## Try This
The Claude Code v2.1.117 fix for Opus 4.7's context window miscalculation (200K vs. 1M) means sessions were autocompacting far too early — burning tokens and losing context on long-running tasks. If you've been hitting unexpected `/compact` prompts or noticing Opus 4.7 "forgetting" earlier work, update now and re-run any sessions that felt truncated. While you're at it, explicitly set effort level in your project config: with default effort now bumped to `high` for Pro/Max on Opus 4.6 and Sonnet 4.6, check whether your task mix actually warrants that — high effort on routine tasks will eat your weekly token budget faster than before.
