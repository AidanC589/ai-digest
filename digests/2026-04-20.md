# AI Digest — 2026-04-20

## TL;DR
Uber burned through its entire AI tooling budget (dominated by Claude Code) just months into 2026, signaling that adoption costs are outpacing productivity gains at scale. Claude Opus 4.7's new tokenizer means roughly 40% higher costs for text workloads — worth checking before you assume pricing parity with 4.6.

## General AI Developments
- [CEOs admit AI had no impact on employment or productivity](https://fortune.com/article/why-do-thousands-of-ceos-believe-ai-not-having-impact-productivity-employment-study/) — A February NBER study of 6,000 executives found nearly 90% saw no productivity or employment impact from AI over three years, with usage averaging just 1.5 hrs/week — Solow's productivity paradox redux. Individual productivity gains in controlled studies (MIT's ~40%) aren't showing up in macro data yet.

## Engineering & AI Workflows
- [Uber's Anthropic AI push hits a wall](https://finance.yahoo.com/sectors/technology/articles/ubers-anthropic-ai-push-hits-223109852.html) — Uber exhausted its AI tooling budget months into 2026 after encouraging engineers to use Claude Code and Cursor via internal usage leaderboards. Claude Code became dominant while Cursor plateaued; Uber is now trialing OpenAI Codex as a cost hedge. 11% of live backend code updates are now AI-written — a real adoption benchmark to compare against.

- [Claude Token Counter, now with model comparisons](https://simonwillison.net/2026/Apr/20/claude-token-counts/#atom-everything) — Opus 4.7's updated tokenizer inflates text token counts ~1.46x over 4.6, making it effectively ~40% more expensive despite identical per-token pricing. High-res images get hit harder (3x), but small images and PDFs are much closer to parity. Worth running your own prompts through the comparison tool before committing to 4.7 for cost-sensitive workloads.

- [Is anyone getting real coding work done with Qwen3.6-35B-A3B-UD-Q4_K_M on a 32GB Mac?](https://www.reddit.com/r/LocalLLaMA/comments/1sq94qx/is_anyone_getting_real_coding_work_done_with/) — Practical field report: Qwen3.6-35B-A3B on 32GB RAM hits a hard wall at 32K context, losing task state on the second compaction pass. The model's own card recommends 128K minimum for complex tasks — a useful constraint to know if you're evaluating local models as a Claude Code cost-reduction strategy.

- [YSK: If you use Claude on your company's Enterprise plan, your employer can access every message you've ever sent, including "incognito" chats](https://www.reddit.com/r/ClaudeAI/comments/1spsugm/ysk_if_you_use_claude_on_your_companys_enterprise/) — Claude's Compliance API lets enterprise admins programmatically pull full chat history, uploaded files, and activity logs — including "incognito" sessions. If you're putting sensitive code, credentials, or architecture details into Claude at work, assume it's logged and accessible to your org.

- [Headless everything for personal AI](https://simonwillison.net/2026/Apr/19/headless-everything/#atom-everything) — Salesforce and others are shifting to API/MCP/CLI-first surfaces explicitly for AI agent access. The practical implication: evaluating tools for your AI workflow should now include "does it have an API or MCP endpoint?" as a first-class criterion, not an afterthought.

## Try This
Before migrating any Claude Code workflow from Opus 4.6 to 4.7, paste your actual system prompt and a representative large context into [Simon Willison's Claude Token Counter](https://simonwillison.net/2026/Apr/20/claude-token-counts/#atom-everything) and run the side-by-side comparison. The 1.46x text inflation means a prompt that cost $5 on 4.6 now costs ~$7.30 on 4.7 — multiply that across thousands of agentic calls and you may want to reserve 4.7 specifically for tasks that need its improved vision or reasoning, while keeping 4.6 for high-volume, text-heavy automation.
