## Tool Updates
- [Claude Code v2.1.207](https://github.com/anthropics/claude-code/releases/tag/v2.1.207) — Major security fix: `${user_config.*}` interpolation in shell-form plugin hook/monitor commands is now rejected to prevent shell injection; migrate to exec form (args array) or `$CLAUDE_PLUGIN_OPTION_<KEY>` env vars. Also fixes: terminal freezing during long streaming responses, spurious prompt-injection warnings, auto-updater overwriting custom launcher scripts, a crash loop in agent teams from malformed mailbox messages, and Bedrock repeatedly requesting fresh AWS SSO credentials. Auto mode is now on by default for Bedrock, Vertex, and Foundry. Default model on Bedrock/Vertex changed to Claude Opus 4.8.

## Try This
The Claude Code v2.1.207 security fix is a prompt to audit any existing plugin hooks you have: open your hook configs now and check whether any shell-form commands use `${user_config.*}` interpolation — these will now be rejected and could break your workflows silently. Migrate them to exec form (an args array instead of a shell string) or switch to reading values via `$CLAUDE_PLUGIN_OPTION_<KEY>` environment variables inside the script. Do this before the next session where Claude Code auto-updates.
