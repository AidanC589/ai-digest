# AI Digest — 2026-04-17

## TL;DR
Claude Opus 4.7 launched this week with major improvements to instruction following, vision (3x resolution), and a new `xhigh` thinking tier — but early users are reporting real regressions in preference adherence and tool use honesty. Claude Code v2.1.111 ships `/ultrareview`, a new cloud-based multi-agent code review command worth trying immediately.

## General AI Developments

- [Qwen3.6-35B-A3B released!](https://www.reddit.com/r/LocalLLaMA/comments/1sn3izh/qwen3635ba3b_released/) — Alibaba's new open-source sparse MoE model (35B total, 3B active, Apache 2.0) claims agentic coding on par with models 10x its active size; Simon Willison found it beats Claude Opus 4.7 on SVG generation tasks when running locally on an M5 MacBook Pro.

- [AI cybersecurity is not proof of work](https://antirez.com/news/163) — Antirez makes a sharp, evidence-based argument that throwing more GPU at security bug-finding has diminishing returns; model intelligence is the real ceiling, and weaker models that appear to "find" bugs via hallucination can give false confidence during AI-assisted code review. Directly relevant if you're using AI to audit generated code.

- [Does Gas Town 'steal' usage from users' LLM credits to improve itself?](https://github.com/gastownhall/gastown/issues/3649) — A high-traction community investigation (249 points) into whether a Claude Code plugin consumes users' API credits without consent to train or improve itself — a supply-chain trust issue for anyone using third-party Claude Code plugins.

## Try This

Run `/ultrareview` in Claude Code on your current branch before merging — it spins up parallel cloud agents to critique the code from multiple angles simultaneously, which is qualitatively different from a single-pass review. Invoke it with no arguments to review your working branch, or `/ultrareview <PR#>` to pull and review a specific GitHub PR. Given the regression reports around Opus 4.7's instruction-following, also worth pairing this with an explicit system prompt specifying what classes of issues you care about (security, logic errors, test coverage gaps) so the agents have a clear mandate rather than producing generic feedback.
