## Tool Updates
- [Claude Code v2.1.187](https://github.com/anthropics/claude-code/releases/tag/v2.1.187) — Security addition: `sandbox.credentials` setting now blocks sandboxed commands from reading credential files and secret env vars. Key fixes: MCP tool calls that hung indefinitely now abort after 5 minutes (configurable via `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`); org-level model restrictions now enforced in the model picker; `--resume` fixed for runs that produced no model turns; structured output no longer loops indefinitely after a successful call; CJK paste mojibake fixed; leaked agent worktree registrations now cleaned up automatically.
- [Codex rust-v0.143.0-alpha.10 through alpha.14](https://github.com/openai/codex/releases/tag/rust-v0.143.0-alpha.14) — Five rapid alpha releases this cycle; no substantive changelogs published — active development but nothing concrete to act on yet.

## Try This
The new `sandbox.credentials` setting in Claude Code v2.1.187 is worth enabling immediately if you run any sandboxed agentic tasks: add `sandbox.credentials: block` to your Claude Code config to prevent sandboxed commands from reading credential files or secret environment variables. This matters most when running multi-step agent workflows that touch your filesystem — it closes a real attack surface where a compromised or hallucinating subagent could exfiltrate API keys or tokens. Check the [release notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.187) for the exact config syntax and pair it with a review of which env vars your current sandbox sessions can currently see.
