## General AI Developments

- **[[AINews] Fable and Mythos officially too dangerous to release](https://www.latent.space/p/ainews-fable-and-mythos-officially)** — The US government issued an emergency export control directive forcing Anthropic to disable Fable 5 and Mythos 5 for all customers worldwide, citing a jailbreak that essentially amounted to "ask the model to find and fix vulnerabilities in a codebase." Anthropic disputes the severity, noting GPT-5.5 has equivalent capability used daily by security defenders. The practical takeaway for infrastructure teams: closed frontier APIs carry explicit geopolitical risk and can vanish overnight — this is the first time Artificial Analysis's Intelligence Frontier chart has moved backward.

- **[Diffusion Gemma is 4x faster, but makes 6x more mistakes!](https://www.reddit.com/r/LocalLLaMA/comments/1u4bne8/diffusion_gemma_is_4x_faster_but_makes_6x_more/)** — Benchmarked on an H100, DiffusionGemma 26B hits 763 tok/s vs Gemma4's 218 tok/s, but error rate jumps from 5 wrong facts to 28 out of ~60 claims, with hallucinated names, colleagues, and prices. The architectural reason: diffusion models optimise for surface smoothness, not factual consistency. Don't use it for code or fact-sensitive generation.

## Engineering & AI Workflows

- **[[AINews] Fable and Mythos — coding agent eval reshuffling](https://www.latent.space/p/ainews-fable-and-mythos-officially)** — Artificial Analysis replaced SWE-Bench Pro with DeepSWE in its Coding Agent Index because SWE-Bench Pro had become gameable via repository history leakage; DeepSWE writes tasks from scratch. New rankings: Claude Code + Fable 5 scored 77, Codex + GPT-5.5 scored 76, Claude Code + Opus 4.8 scored 73. Also notable: Claude Code underperformed other harnesses on the same underlying model, suggesting the eval harness quality is now a first-class variable when interpreting benchmarks.

- **[Slightly reducing the sloppiness of AI generated frontend](https://envs.net/~volpe/blog/posts/reduce-slop.html)** — A practical finding: prompting AI agents to generate UI "in the style of a Qt app" consistently removes the generic slop aesthetic compared to vague style requests or mimicking named web design trends. The author has applied this across multiple personal tools using Codex. Worth testing as a system prompt addition for any frontend generation workflow.

- **[Statement on the US government directive to suspend access to Fable 5 and Mythos 5](https://simonwillison.net/2026/Jun/13/us-government-directive-to-suspend-access/#atom-everything)** — Simon Willison documented the Fable 5 cutoff in real time with a polling script, confirming API access died at 6:59pm PT on June 12. Practical note: if your workflow or CI pipeline depends on a specific named model endpoint, you need a fallback model alias and automated detection for 404 failures — this incident showed a frontier model can go from available to gone in under two hours.

## Tool Updates

- **[Claude Code v2.1.176](https://github.com/anthropics/claude-code/releases/tag/v2.1.176)** — Large release. Key fixes for AI-assisted workflows: hook `if` conditions for `Read`/`Edit`/`Write` tool paths (e.g. `Edit(src/**)`) now match correctly after being documented but broken; `availableModels` allowlist enforcement is fixed so alias models can't be silently redirected to blocked models via env vars; auto mode fallback for Fable 5 unavailability now works correctly. Also fixes tmux clipboard copy, Remote Control model-switching, background session stalls, and Windows daemon startup issues. Session titles now generate in the language of your conversation.
