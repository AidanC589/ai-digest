## Engineering & AI Workflows

- [If AI writes your code, why use Python?](https://medium.com/@NMitchem/if-ai-writes-your-code-why-use-python-bf8c4ba1a055) — The argument is concrete: Rust's tight compiler feedback loop lets AI agents self-correct in real time using error messages as training signals, making it easier for AI to write correct Rust than correct C++. Real evidence includes a 100K-line C compiler in Rust (~$20K in Claude Code sessions) and a 25K-line JS engine port completed in two weeks.

- [Quoting James Shore on AI maintenance costs](https://simonwillison.net/2026/May/11/james-shore/#atom-everything) — A sharp warning: if AI doubles your code output but doesn't halve maintenance costs, you've permanently doubled your burden. The implication for reviewers — don't just approve AI code for correctness, actively evaluate whether it's *simpler and easier to maintain* than what you'd have written manually.

## Tool Updates

- [Claude Code v2.1.139](https://github.com/anthropics/claude-code/releases/tag/v2.1.139) — Major release. Highlights: **Agent View** (Research Preview) gives a single dashboard of all running/blocked/done sessions — run `claude agents` to start dispatching multiple parallel sessions without terminal tabs. New **`/goal` command** lets you set a completion condition and Claude keeps working across turns until it's met, with a live overlay showing elapsed time, turns, and tokens. Also added: `/scroll-speed` tuning, `claude plugin details <name>` with per-session token cost projection, hook `args: string[]` for shell-safe execution, `continueOnBlock` for PostToolUse hooks to feed rejection reasons back to Claude, and `CLAUDE_PROJECT_DIR` in MCP stdio server environments. Notable fixes include a credential deadlock, unbounded SSE memory growth, and broken wildcard Skill permissions.

- [OpenAI Codex rust-v0.131.0-alpha.6 through alpha.9](https://github.com/openai/codex/releases/tag/rust-v0.131.0-alpha.9) — Four alpha releases in quick succession; no changelogs published beyond version bumps, suggesting active internal iteration.

## Try This

Use Claude Code's new `/goal` command to offload long multi-step tasks entirely: open a session, type `/goal` followed by a concrete, verifiable completion condition (e.g., `/goal all tests pass and the migration script runs without errors on the staging dataset`), then walk away. Claude will continue working across turns until the condition is met, and the live overlay will show you elapsed time, turns, and token spend so you can judge whether it's converging or spinning. Pair this with the new Agent View (`claude agents`) to monitor it alongside other running sessions without keeping a terminal tab open.
