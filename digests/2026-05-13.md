# AI Digest — 2026-05-13

## TL;DR
Claude Code shipped a substantial bug-fix release (v2.1.140) addressing hangs, Windows event-loop stalls, and symlink regressions. OpenAI is deprecating its finetuning APIs, signalling a broader industry shift toward long prompts and RAG over custom model training. A new open-source tool, Statewright, uses formal state machines to constrain Claude Code and other agents — showing consistent reliability gains without bigger models.

## General AI Developments

- [Amazon employees are "tokenmaxxing" due to pressure to use AI tools](https://arstechnica.com/ai/2026/05/amazon-employees-are-tokenmaxxing-due-to-pressure-to-use-ai-tools/) — Amazon set targets for 80%+ of developers to use AI weekly and published internal token-consumption leaderboards; employees responded by automating fake AI activity to game the metrics. A cautionary case study in how vanity measurement corrodes genuine adoption.

- [[AINews] The End of Finetuning](https://www.latent.space/p/ainews-the-end-of-finetuning) — OpenAI is deprecating its finetuning APIs, and the industry consensus is shifting: for most teams, very long structured prompts (à la Claude's system prompt / Constitution approach) are now outperforming finetuned models, while top-tier shops doing RLFT on open weights are moving in the opposite direction with more investment, not less.

- [Sampling More, Getting Less: Calibration is the Diversity Bottleneck in LLMs](https://arxiv.org/abs/2605.11128) — Across 14 models, diversity collapse in LLM outputs is traced to two specific miscalibration types (order and shape), not sampling strategy. Relevant for anyone relying on high-temperature sampling to get varied AI-generated code solutions — the bottleneck is in the model distribution itself.

## Engineering & AI Workflows

- [Show HN: Statewright – Visual state machines that make AI agents reliable](https://github.com/statewright/statewright) — Instead of using bigger models or longer prompts to fix agent reliability, Statewright constrains tool access and valid transitions per-state using a Rust-enforced state machine, integrated with Claude Code via MCP. The author reports consistent improvements across model families above 13B params, including Haiku and Sonnet punching above their weight — worth trialling on multi-step agentic workflows.

- [llm 0.32a2](https://simonwillison.net/2026/May/12/llm/#atom-everything) — Simon Willison's `llm` CLI now routes GPT-5-class models through `/v1/responses` instead of `/v1/chat/completions`, enabling interleaved reasoning across tool calls and surfacing summarised reasoning tokens in a distinct colour. Useful for debugging what a model is actually "thinking" before it produces code.

- [datasette 1.0a29](https://simonwillison.net/2026/May/12/datasette/#atom-everything) — Willison used Codex CLI with GPT-5.5 xhigh to build a minimal Dockerfile that reproduced a gnarly race-condition segfault — a practical example of using AI to construct targeted reproduction cases rather than just fix code, which is a high-leverage prompt pattern for hard bugs.

- [CSP Allow-list Experiment](https://simonwillison.net/2026/May/13/csp-allow/#atom-everything) — A sandboxed iframe experiment where a custom `fetch()` intercepts CSP errors and prompts the user to add domains to an allow-list, built entirely with GPT-5.5 xhigh in the Codex desktop app. Good reference for the kind of security-adjacent browser plumbing that AI can now handle end-to-end with light steering.

- [Needle: We Distilled Gemini Tool Calling Into a 26M Model](https://www.reddit.com/r/LocalLLaMA/comments/1tb9b0r/needle_we_distilled_gemini_tool_calling_into_a/) — A 26M parameter model running at 6000 tok/s handles single-shot function calling by removing FFN layers entirely (attention + gating only), because tool calling is retrieval-and-assembly, not reasoning. If you're building agents where tool dispatch is a bottleneck or cost concern, this architecture is worth understanding.

## Tool Updates

- [Claude Code v2.1.140](https://github.com/anthropics/claude-code/releases/tag/v2.1.140) — Multiple fixes relevant to daily use: `/goal` no longer silently hangs when hooks are restricted; settings hot-reload no longer misfires on symlinked config files; `claude --bg` stops dropping connections on idle-exit; Windows users get relief from an event-loop stall caused by repeated `where.exe` spawns for missing executables (e.g. `gh`). Also: plugins now warn visibly (in `/doctor` and `claude plugin list`) when a default component folder is silently ignored due to `plugin.json` misconfiguration.

- [Codex rust-v0.131.0-alpha.10 / alpha.11](https://github.com/openai/codex/releases/tag/rust-v0.131.0-alpha.11) — Two alpha releases shipped with no detailed changelogs; worth keeping an eye on the repo if you're on the Rust-based Codex CLI.

## Try This

Install the Statewright plugin into Claude Code right now and run it against a real bug or feature task: `/plugin marketplace add statewright/statewright`, then `/plugin install statewright`, then `/reload-plugins`, and finally say "start the bugfix workflow." Watch which tools Claude is offered at each phase versus its normal unconstrained behaviour — the contrast between 5 scoped tools and the full palette is immediately instructive for understanding where your current prompts are bleeding context and inviting scope creep.
