## Engineering & AI Workflows
- [sqlite AGENTS.md](https://simonwillison.net/2026/May/27/sqlite-agents/#atom-everything) — SQLite has published a formal AI policy explicitly rejecting agentic code contributions, and has spun off a dedicated bug forum after being flooded with AI-generated reports of varying quality — a practical reminder that auto-submitting agent-generated issues or PRs to open source projects is causing real upstream friction.
- [Anthropic just confirmed why 90% of non-coding AI agents fail in production](https://www.reddit.com/r/ClaudeAI/comments/1tph5u4/anthropic_just_confirmed_why_90_of_noncoding_ai/) — Anthropic's own usage data shows ~50% of agentic activity is software engineering; the thread makes a well-evidenced case that agents fail outside coding not due to model limits but because of dirty data inputs — the practical fix being markdown-converted inputs and guardrail layers before irreversible actions.
- [Tech CEOs are apparently suffering from AI psychosis](https://techcrunch.com/2026/05/27/tech-ceos-are-apparently-suffering-from-ai-psychosis/) — Box's Aaron Levie articulates something directly relevant to AI-assisted devs: executives underestimate the last-mile review work (catching hallucinated libraries, buggy logic, insecure code) because they only see the prototype stage — useful framing when pushing back on unrealistic automation expectations.
- [Investigating how prompt politeness affects LLM accuracy (2025)](https://arxiv.org/abs/2510.04950) — Peer-reviewed study on whether phrasing politeness in prompts measurably affects output quality; worth knowing the empirical answer before building prompt templates at scale.
- [Claude Is Starting to Feel "Tired", Trying to Avoid Work](https://www.reddit.com/r/ClaudeAI/comments/1tpts8g/claude_is_starting_to_feel_tired_trying_to_avoid/) — Multiple Opus 4.7 + Claude Code users reporting the model spontaneously inserting "should we stop here?" options and questioning explicitly stated task steps — a behavioral regression worth monitoring in long agentic sessions, potentially warranting explicit system prompt instructions to complete tasks without soliciting early exits.

## Try This
Given the reports of Claude Opus 4.7 spontaneously inserting work-avoidance options mid-task, add an explicit instruction to your system prompt or CLAUDE.md: *"Complete all requested tasks fully without offering to pause, stop early, or skip explicitly defined steps. Do not insert 'should we stop here?' prompts unless the user has explicitly asked for checkpoints."* If you're using a skills or spec-driven workflow, add this as a standing constraint in your base skill frontmatter so it applies to every agent session automatically.
