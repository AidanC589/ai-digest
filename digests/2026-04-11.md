## Engineering & AI Workflows
- [[AINews] AI Engineer Europe 2026](https://www.latent.space/p/ainews-ai-engineer-europe-2026) — The "cheap executor + expensive advisor" pattern is crystallising into a standard architecture: use a fast model for most steps, escalate hard decisions to a smarter model. Concrete evidence: Haiku + Opus more than doubles BrowseComp score vs Haiku alone, and Sonnet + Opus improves SWE-bench Multilingual while *reducing* task cost — worth building into your own agent harnesses now.
- [[AINews] AI Engineer Europe 2026](https://www.latent.space/p/ainews-ai-engineer-europe-2026) — Practitioners are hitting a real pain point: top models are specialised (Opus wins on frontend/agentic flows, GPT-5.4 wins on backend/distributed systems), but tools like Claude Code and Codex remain provider-bound with no automatic routing. Expect pressure on tooling vendors to support shared context + cross-model collaboration in a single workflow.
- [[AINews] AI Engineer Europe 2026](https://www.latent.space/p/ainews-ai-engineer-europe-2026) — Qwen Code v0.14.x shipped sub-agent model selection and planning mode directly in the product, making model-mixing explicit at the tool level rather than requiring external harness code — a sign that orchestration primitives are moving into the tools themselves.

## Tool Updates
- [Claude Code v2.1.101](https://github.com/anthropics/claude-code/releases/tag/v2.1.101) — **Security fix**: a command injection vulnerability in the POSIX `which` fallback used by LSP binary detection has been patched — update immediately. Also adds `/team-onboarding` to generate teammate ramp-up guides from your local usage, OS CA certificate store trust by default (enterprise TLS proxies now work without extra setup), and fixes a hardcoded 5-minute request timeout that was aborting slow local LLMs and extended thinking runs regardless of `API_TIMEOUT_MS`.
- [Claude Code v2.1.101](https://github.com/anthropics/claude-code/releases/tag/v2.1.101) — `permissions.deny` rules now correctly override a `PreToolUse` hook's `permissionDecision: "ask"` — previously a hook could silently downgrade a deny to a prompt, a meaningful security/governance gap for teams using hooks for access control.
- [Codex 0.120.0](https://github.com/openai/codex/releases/tag/rust-v0.120.0) — Background agent progress can now stream in real-time via Realtime V2 while work is still running, with follow-up responses queued until the active response completes. Hook activity in the TUI is also cleaner, with live hooks shown separately from completed output.

## Try This
Test the advisor pattern on your next complex agent task: instead of routing everything through a single capable model, chain a fast model (e.g. Haiku or a smaller local model) for routine steps and explicitly escalate to Opus or equivalent only when you hit a decision point requiring reasoning or judgment. In Claude Code, you can approximate this by using a lighter model in your sub-agent config and reserving the primary model for planning mode. The Latent Space recap reports this approach more than doubles performance on hard tasks while cutting cost — try it on a real codebase task this week and compare outputs directly.
