- [5 Trends That Defined AI Engineering at World's Fair 2026](https://www.latent.space/p/aiewf26trends) — The field has formally shifted from agents-as-product to harness engineering: the system surrounding the model (context management, permissions, evals, persistent state) is now considered equally important as the model itself. Complete agent autonomy is out; human-in-the-loop augmentation is the consensus.

- [[AINews] not much happened today](https://www.latent.space/p/ainews-not-much-happened-today-c72) — Two actionable takeaways: (1) stale `agents.md` instruction files can act as self-inflicted prompt injection causing multi-hour stalls in long-running tasks — audit yours regularly; (2) LangSmith now traces Claude Code, Cursor, Copilot, and OpenCode, exposing tool calls, subagents, and token usage for observability.

## Tool Updates

- [Claude Code v2.1.210](https://github.com/anthropics/claude-code/releases/tag/v2.1.210) — Major release with a security-critical fix: the Agent tool has been **hardened against indirect prompt injection via content a subagent read** — this is directly relevant to anyone running unattended or multi-agent sessions. Also fixed: `isolation: 'worktree'` subagents could previously run git-mutating commands against the main repo (now fixed); hook callback timeouts were being misreported as user rejections, causing unattended sessions to stall; and `ultracode` keyword was incorrectly firing on webhook payloads and PR comments. New: live elapsed-time counter on long-running tool calls, startup warnings for deprecated `Write(path)` permission rules (use `Edit(path)` or `Read(path)`).

- [Claude Code v2.1.209](https://github.com/anthropics/claude-code/releases/tag/v2.1.209) — Patch fixing `/model` and other dialogs being blocked in background agent sessions.

- [GitHub Changelog: Dependabot 3-day cooldown now default](https://simonwillison.net/2026/Jul/14/github-changeling/#atom-everything) — Dependabot now waits at least three days after a new package release before opening a version update PR, reducing noise from immediately-yanked or buggy releases. No configuration required.

- [Codex 0.145.0-alpha.10/11/12](https://github.com/openai/codex/releases/tag/rust-v0.145.0-alpha.12) — Three rapid alpha releases with no public changelogs; active iteration in progress.

## Try This

Audit your `agents.md` (or equivalent instruction file) for staleness right now — per the AINews recap, outdated agent instructions are being flagged as a form of self-inflicted prompt injection that can cause multi-hour stalls in long-running Claude Code or Codex sessions. Open the file, add a `## Last reviewed` datestamp at the top, and walk through each instruction asking: "Does this still reflect how the codebase actually works?" Delete anything that no longer applies. Then set a calendar reminder to repeat this monthly, or hook it into your project's onboarding checklist so new context gets added when the system changes.
