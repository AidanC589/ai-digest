## Try This
After your AI assistant writes a new function or feature, explicitly prompt it: *"Use the CLI / a scratch script to manually exercise this implementation — run it, observe the output, and look for edge cases the unit tests don't cover."* Willison did exactly this in sqlite-utils 4.1 with `uv run python -c`, and it caught two bugs that the automated test suite missed entirely. The key is to ask the model to *run* its own code in a real environment rather than just reason about correctness — you get a second, empirical review pass at near-zero cost.
