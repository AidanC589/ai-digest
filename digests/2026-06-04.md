- [RAG label framing shifts LLM adoption of wrong answers by 56–84 percentage points](https://arxiv.org/abs/2606.04109) — a controlled study across GPT-5.5, DeepSeek V4 Pro, and others shows that wrapping context with `Instruction:` or `Reference:` dramatically increases the model's adoption of injected misinformation, while `Example:` suppresses it. Direct implication for anyone building RAG pipelines or prompt templates that feed AI coding assistants with retrieved docs.
- [RAG yields only 1–2 point improvements in biomedical QA; backbone model choice matters far more](https://arxiv.org/abs/2606.04127) — across 10 datasets and 4 retrieval methods, retrieval barely moved the needle; the model's ability to use evidence was the bottleneck. Applies broadly: if your AI code review pipeline retrieves docs or examples, the model you choose will dominate retrieval quality.

## Try This
The discourse-label paper above is immediately actionable for your RAG and prompt templates: swap any context wrapper labelled `Reference:` or `Instruction:` to `Example:` when you want the model to *consider* retrieved content without over-weighting it — for instance, when feeding Claude Code a retrieved code snippet as a pattern to follow rather than a binding rule. Conversely, use `Instruction:` only when you explicitly want the model to adopt the content. Audit your existing system prompts and MCP server descriptions for unintentional binding labels that may be causing the model to over-trust injected context and hallucinate less carefully.
