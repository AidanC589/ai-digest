## Try This
Use Anthropic's newly documented `srt` (Sandbox Runtime) to audit what your Claude Code sessions can actually reach. Spin up a session with `srt` enabled, run a task that touches credentials or network calls, then inspect the egress logs to verify that the documented controls are actually blocking what they claim. This is more valuable than trusting the docs — the previously missed `api.anthropic.com/v1/files` exfiltration vector (now patched) is a reminder that sandbox boundaries are worth empirically verifying rather than assumed. See [the Anthropic sandboxing overview](https://simonwillison.net/2026/May/30/how-we-contain-claude/#atom-everything) for the documented architecture before you start.
