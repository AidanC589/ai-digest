## Tool Updates
- [Claude Code v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195) — Key fix: hook matchers with hyphenated identifiers (e.g. `code-reviewer`, `mcp__brave-search`) were substring-matching incorrectly; they now exact-match. Use `mcp__brave-search__.*` to match all tools from a hyphenated MCP server. Also fixes background agent data loss when written by a newer version, and improves remote session startup with a provisioning checklist.

## Try This
If you use Claude Code with any MCP servers or hooks that have hyphenated names, audit your `.claude/settings.json` immediately after upgrading to v2.1.195 — the old substring-matching behaviour may have been silently activating unintended hooks. Then update any broad matchers to use the explicit wildcard form: for example, change `mcp__brave-search` to `mcp__brave-search__.*` to correctly match all tools from that server rather than relying on the previously broken (and now fixed) partial-match behaviour.
