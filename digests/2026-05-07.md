## Engineering & AI Workflows
- [Vibe coding and agentic engineering are getting closer than I'd like](https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/#atom-everything) — Simon Willison admits he's stopped reviewing every line Claude Code writes for production, because routine tasks (JSON API + SQL + tests) are reliably correct — but flags the guilt and risk this creates; his framing: the goal must be "higher quality faster," not just "faster," and peer review norms from larger orgs may be the right mental model for trusting AI output.

- [Live blog: Code w/ Claude 2026](https://simonwillison.net/2026/May/6/code-w-claude-2026/#atom-everything) — Key practitioner advice from Anthropic's event: teams getting the most from Claude focus on automated evals, simple scaffolding, and imaginative use cases; the "advisor strategy" (Sonnet calling Opus for advice on demand) delivered frontier-model quality at 5x lower cost for one customer.

- [New in Claude Managed Agents: dreaming, outcomes, multiagent orchestration, and webhooks](https://www.reddit.com/r/ClaudeAI/comments/1t5j84j/new_in_claude_managed_agents_dreaming_outcomes/) — Three new platform features now in beta: "Dreaming" (scheduled review of past sessions to curate agent memory — Harvey saw ~6x higher task completion rates), "Outcomes" (rubric-based quality grading with automatic iteration until passing), and multi-agent orchestration (lead agent delegates to parallel specialists); dreaming is research preview, the others are public beta.

- [the part nobody warns you about](https://www.reddit.com/r/ClaudeAI/comments/1t5vs8t/the_part_nobody_warns_you_about/) — A widely-resonant post on the hidden cost of AI-generated codebases: 800-line functions, ambiguous variable names, undocumented decisions — the build phase is fast, the debugging phase is where the debt lands; worth internalising as a risk when steering agents without enforcing code style and size constraints upfront.

## Try This
Set up Anthropic's new **Outcomes** feature in Claude Managed Agents this week: write a rubric for one of your agent's recurring tasks (e.g. "API endpoint must include input validation, error handling, and passing tests; output must be under 200 lines"), wire it to the grader, and let the agent iterate until it meets the bar rather than accepting first-pass output. This directly addresses the "800-line handleStuff function" failure mode — you're encoding your quality standards as an automated check rather than relying on manual review.
