## Engineering & AI Workflows
- [Claude Opus 4.8: "a modest but tangible improvement"](https://simonwillison.net/2026/May/28/claude-opus-4-8/#atom-everything) — The headline engineering improvement: Opus 4.8 is ~4x less likely to let code flaws pass unremarked, actively flags uncertainties rather than confidently hallucinating, and now supports mid-conversation system messages — letting you inject updated instructions mid-session without busting your prompt cache on earlier turns.
- [[AINews] Anthropic raises $965B Series H, releases Opus 4.8 and Dynamic Workflows/ultracode](https://www.latent.space/p/ainews-anthropic-raises-965b-series) — Dynamic Workflows (also called "ultracode") in Claude Code is the bigger long-term story: it lets Claude plan a task, then fan out across hundreds of parallel subagents — already used to rewrite 750k LOC of Bun from Zig to Rust in 6 days. This changes the calculus for large refactors and migrations.
- [The Age of Async Agents — Cognition's Walden Yan & OpenInspect's Cole Murray](https://www.latent.space/p/cognition) — Practical framing of where agent workflows are heading: the shift is from "developer in the loop watching a terminal" to "developer as orchestrator reviewing async agent output." The key insight is that task-harness fit now matters as much as model quality — bespoke vertical harnesses outperform generic ones by narrowing tools, prompts, and context to the task.
- [AI sticker shock hits corporate America](https://www.axios.com/2026/05/28/ai-spending-roi-enterprise-costs) — Enterprises are hitting unexpected cost cliffs with AI; one Anthropic client reportedly spent $500M in a single month after failing to set usage limits on Claude licenses. ## Try This
Claude Opus 4.8's new mid-conversation system messages are immediately useful for long agentic sessions: instead of restating your full system prompt when context drifts, inject a `role: "system"` message mid-conversation with a corrective instruction (e.g., "Reminder: all output must use TypeScript strict mode and never use `any`"). This preserves prompt cache hits on earlier turns, cuts input costs on long loops, and lets you steer without breaking the session — try it in your next Claude Code session by passing an updated system message after a tool-use turn where the model has started going off-spec.
