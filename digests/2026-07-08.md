- [Automating AI Away](https://replicated.live/blog/away) — A practical account of sandwiching LLM non-determinism between deterministic tools: fast formal tools on one side, strict workflow definitions on the other. Key insight: when Claude repeats a mistake, automate that step away; when it fails verification, automate the check. Direct relevance: prompts that include prior AI-generated answers or error examples risk contaminating model reasoning even without framing them as authoritative sources.

## Try This
Use the "guardrails on re-entry" pattern from the slopfix team: after any significant AI-built feature, prompt Claude Code to write a short CLAUDE.md entry capturing the architectural decisions made and the patterns that must not be violated (e.g. "date formatting always uses `formatDate()` in `utils/dates.ts`, never inline"). Instruct it to also generate one lint rule or CI check that enforces the most likely regression point. Run this as a closing step on every session — it compounds over time into a self-documenting constraint system that future AI sessions inherit automatically.
