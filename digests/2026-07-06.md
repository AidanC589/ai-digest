## Try This
Use the `openai/codex-plugin-cc` integration to add a second-opinion code review step to your Claude Code workflow: after Claude Code generates a feature, invoke Codex through the plugin with the prompt "Review this diff for security vulnerabilities, logic errors, and cases where the implementation doesn't match the stated intent — be specific and cite line numbers." Two models reviewing the same output catch different classes of errors, and the Codex/Claude pairing specifically has different training biases, making disagreements between them a reliable signal that something needs closer human attention.
