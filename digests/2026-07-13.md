## Try This
Audit your eval harness for incumbent bias before benchmarking any new model. As Ploy discovered, budget limits, file read assumptions, and tool-call patterns silently favour whichever model you built around first. Before your next cross-model comparison, run a sample of "failures" from the challenger model through full traces and triage whether they failed on task or failed because your harness expected the incumbent's behavior — then fix those before trusting any pass rates. See the [Ploy migration writeup](https://ploy.ai/blog/migrating-a-production-ai-agent-to-gpt-5-6) for a concrete checklist of what to check.

## Token Saving Techniques
- [Migrating a production AI agent to GPT-5.6: 2.2x faster, 27% cheaper](https://ploy.ai/blog/migrating-a-production-ai-agent-to-gpt-5-6) — Switching providers requires re-implementing prompt caching from scratch since cache mechanics are provider-specific; GPT-5.6 also uses batched parallel tool calls by default (vs. Claude's sequential style), which can reduce total turns and lower costs if your tool schemas support it.
- [AgentKGV: Agentic LLM-RAG Framework with Two-Stage Training for the Fact Verification of Knowledge Graphs](https://arxiv.org/abs/2607.09092) — GRPO-based training cut average search calls from 3.24 to 1.63 per query without accuracy loss — a useful signal that optimising agent search policy (not just prompts) can halve retrieval costs at scale.
