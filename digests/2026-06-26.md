## Tool Updates

- [Claude Code v2.1.193](https://github.com/anthropics/claude-code/releases/tag/v2.1.193) — Substantial release: new `autoMode.classifyAllShell` setting routes all Bash/PowerShell through the auto-mode classifier (not just arbitrary-code patterns); live file path autocomplete in bash mode; auto-denial reasons now appear in transcripts and `/permissions`; automatic memory-pressure reaping for idle background shells; several background agent bugs fixed including a phantom subagent re-running the main conversation. **Note:** `claude_code.assistant_response` OpenTelemetry log event is now emitted by default if you already log user prompts — set `OTEL_LOG_ASSISTANT_RESPONSES=0` to avoid inadvertently logging response content.

- [OpenAI Codex rust-v0.143.0-alpha.22 through alpha.25](https://github.com/openai/codex/releases/tag/rust-v0.143.0-alpha.25) — Four alpha releases this week; changelogs are minimal but indicates active iteration on the Rust rewrite.

## Try This

Add a `DESIGN.md` file to your repo root this week and reference it explicitly at the start of any UI-related Claude Code session: "Read DESIGN.md before making any visual changes." The [google-labs-code/design.md spec](https://github.com/google-labs-code/design.md) gives you a structured format for encoding your colour palette, typography, spacing system, and component conventions — and because it's a persistent file in the working directory, Claude Code will pick it up automatically in future sessions without you needing to re-explain your design system each time.
