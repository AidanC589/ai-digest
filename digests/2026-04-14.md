## Tool Updates
- [Claude Code v2.1.105](https://github.com/anthropics/claude-code/releases/tag/v2.1.105) — Major release with several workflow-relevant changes: **PreCompact hook** now lets you block context compaction programmatically (exit code 2); **stalled stream handling** now aborts after 5 minutes and retries rather than hanging forever; **WebFetch strips `<style>`/`<script>` content** so CSS-heavy pages no longer eat your context budget; `/proactive` is now an alias for `/loop`; skill description cap raised from 250 to 1,536 characters. Fixes include dropped images on queued messages, garbled bash output from rich/loguru logging, and MCP tools missing on the first turn of headless sessions — the last two are directly relevant to agent workflows.

- [OpenAI Codex rust-v0.121.0-alpha.4 through alpha.6](https://github.com/openai/codex/releases/tag/rust-v0.121.0-alpha.6) — Three rapid alpha releases with no published changelogs; active churn suggests the Rust rewrite is moving fast but treat as unstable.

## Try This
In Claude Code, set up a **PreCompact hook** to enforce a checkpoint before the context gets compressed on long sessions. Write a hook script that dumps a structured summary of open TODOs, pending file changes, and current working hypothesis to a `session-state.md` file, then exits with code 0 to allow compaction. This gives you a human-readable audit trail and prevents Claude from silently losing task context mid-session — especially useful on multi-hour agentic runs. The new v2.1.105 hook support (`{"decision":"block"}` or exit code 2) also lets you *block* compaction entirely if your hook detects the session is at a critical decision point.
