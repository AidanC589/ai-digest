## Engineering & AI Workflows
- [An AI Vibe Coding Horror Story](https://www.tobwr.ch/an-ai-vibe-coding-horror-story/) — A medical practice used a coding agent to build a patient management system, published it to the internet with all data unencrypted, client-side-only auth, and audio sent to US AI APIs — a 30-minute poke exposed full read/write access. Direct evidence that AI-generated code requires a security-aware reviewer, not just a vibe check.
- [The human cost of 10x: How AI is physically breaking senior engineers](https://techtrenches.dev/p/the-human-cost-of-10x-how-ai-is-physically) — UC Berkeley research and Upwork data confirm AI increases workload for 77% of users; GitHub Octoverse 2025 shows PR volume up 23% YoY and lines of code per developer up 76% in eight months, while the SmartBear/Cisco finding holds: defect detection drops from 87% to 28% as PRs exceed 1,000 lines. If you're reviewing AI output, keep PRs small or you're statistically missing most bugs.
- [TUI to see where Claude Code tokens actually go](https://www.reddit.com/r/ClaudeAI/comments/1skqub5/tui_to_see_where_claude_code_tokens_actually_go/) — `npx codeburn` reads Claude Code's local session transcripts and classifies spend into 13 categories; one heavy user found 56% of cost was plain conversation (no tool use) versus 21% on actual code edits — useful for identifying where to tighten prompts or restructure sessions.
- [Exploring the new `servo` crate](https://simonwillison.net/2026/Apr/13/servo-crate-exploration/#atom-everything) — Simon Willison tasked Claude Code with exploring an unfamiliar Rust crate, building a CLI tool, and probing WebAssembly feasibility; the resulting tool works, demonstrating Claude Code as a practical research accelerator for unfamiliar codebases rather than just a code generator.
- [Steve Yegge](https://simonwillison.net/2026/Apr/13/steve-yegge/#atom-everything) — A viral claim that Google's AI adoption looked like John Deere's drew a sharp rebuttal from Addy Osmani (40K+ SWEs using agentic coding weekly) and Demis Hassabis; the more useful signal is the industry-wide pattern cited — 20% agentic power users, 20% refusers, 60% still on chat tools like Cursor.

## Tool Updates
- [v2.1.105](https://github.com/anthropics/claude-code/releases/tag/v2.1.105) — Large Claude Code release: adds `PreCompact` hook (lets hooks block context compaction), `/proactive` as alias for `/loop`, improved stalled-stream handling (abort after 5 min, retry non-streaming), and `WebFetch` now strips `<style>`/`<script>` content. **Critical regression**: broke terminal login via auth code paste — if you upgraded and can't log in, downgrade to v2.1.104. Fix tracked at [#47745](https://github.com/anthropics/claude-code/issues/47745).
- [v2.1.107](https://github.com/anthropics/claude-code/releases/tag/v2.1.107) — Minor follow-up: thinking hints now appear sooner during long operations.
- [You can now switch models mid-chat](https://www.reddit.com/r/ClaudeAI/comments/1skm9tw/you_can_now_switch_models_midchat/) — Claude.ai now lets you swap models without starting a new conversation, useful for escalating to a more capable model mid-thread when complexity increases.
- [rust-v0.121.0-alpha.7](https://github.com/openai/codex/releases/tag/rust-v0.121.0-alpha.7) — OpenAI Codex CLI continues rapid alpha iteration (alpha.4 through alpha.7 shipped); no release notes beyond version bumps.
