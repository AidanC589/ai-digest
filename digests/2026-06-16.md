## Tool Updates
- [Claude Code v2.1.178](https://github.com/anthropics/claude-code/releases/tag/v2.1.178) — Notable security improvement: subagent spawns are now evaluated by the classifier *before* launch, closing a gap where a subagent could request blocked actions without review. Also adds `Tool(param:value)` permission syntax for fine-grained control (e.g. `Agent(model:opus)` to block Opus subagents), fixes compaction not honoring `--fallback-model`, and fixes several subagent transcript/backgrounding bugs. Multiple MCP and OAuth fixes too.
- [Codex 0.140.0](https://github.com/openai/codex/releases/tag/rust-v0.140.0) — Adds `/import` to selectively pull setup, project config, and recent chats from Claude Code (direct migration path), `/usage` views for token activity, permanent session deletion, and managed Amazon Bedrock auth. Also removed experimental realtime voice controls. Performance improvements for large repos and long sessions.

## Try This
In Claude Code v2.1.178, use the new `Tool(param:value)` permission syntax to lock down which models your subagents can spawn. Add a rule like `Agent(model:opus)` to your disallowedTools list to prevent subagents from escalating to expensive models without explicit approval — especially useful in agentic workflows where a subagent might autonomously decide it needs a more capable model mid-task. Pair this with the new pre-launch classifier check (automatic in this release) to get a tighter review loop before any subagent runs.
