## Tool Updates

- [Claude Code v2.1.181](https://github.com/anthropics/claude-code/releases/tag/v2.1.181) — Significant release. Key additions: `/config key=value` lets you set any setting inline during a session (e.g. `/config thinking=false`). Critical fixes: Write/Edit producing 0-byte or truncated files on network drives and cloud-synced folders is fixed; macOS TUI freeze at session start (when Spotlight is reindexing) is fixed; a ~120ms-per-launch startup regression introduced in 2.1.169 is resolved. Also fixes foreground subagents spawning unbounded nested chains — now capped at 5 levels.

- [OpenAI Codex 0.141.0](https://github.com/openai/codex/releases/tag/rust-v0.141.0) — Remote executors now use authenticated, end-to-end encrypted Noise relay channels. Cross-platform remote execution preserves executor-native working directories and shells. Performance improvements: caching tool search and eliminating repeated request/history copies reduces latency and memory in large tool-heavy sessions. SQLite pinned to a version with the WAL-reset corruption fix.

## Try This
Use the new Claude Code `/config` inline syntax to manage thinking mode per-task without leaving the session. At the start of a deep architectural review, run `/config thinking=true` to enable extended reasoning, then switch to `/config thinking=false` before routine file edits to avoid burning tokens unnecessarily. This is cleaner than toggling settings globally and lets you match compute to task complexity without breaking flow.
