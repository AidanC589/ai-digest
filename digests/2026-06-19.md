## Try This

With Claude Code v2.1.183's new safety guards live, test your existing automation scripts and CI pipelines for any flows that relied on Claude autonomously running `git reset --hard` or infrastructure destroy commands — they will now block and wait for explicit confirmation. More usefully: deliberately phrase your prompts when you *do* want destructive operations, e.g. *"Discard all local changes and reset to HEAD"* or *"Destroy the staging stack in Terraform"* — the new guards are keyed to whether you explicitly asked, so precise language in your prompts is now load-bearing for automation to proceed without interruption.
