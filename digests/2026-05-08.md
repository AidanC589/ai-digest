## Try This
If you're doing security-adjacent code review with Claude, take the Mozilla approach and **stack filters**: first prompt Claude to find potential vulnerabilities in a file, then pipe those findings into a second Claude call that acts as a skeptical reviewer tasked with discarding false positives — explicitly instructing it to flag anything that Firefox's existing defense-in-depth would already block. This two-pass pattern (generate signal, then filter noise) is what Mozilla credits for making AI security reports actually useful rather than a burden on maintainers.
