## Engineering & AI Workflows

- [Show HN: 10x better performance from the Coding Harnesses with LLM-wiki](https://llm-wiki.net/) — Ships as a native Claude Code plugin, Codex plugin, or `AGENTS.md` file; spins up 5–10 parallel agents to research a topic and compile it into a persistent, cross-referenced Markdown wiki. Relevant for anyone using Claude Code on large, unfamiliar codebases where building institutional knowledge matters across sessions.

- [Temporary Cloudflare Accounts for AI agents](https://simonwillison.net/2026/Jun/21/temporary-cloudflare-accounts/#atom-everything) — `npx wrangler deploy --temporary` now deploys a Cloudflare Workers project for 60 minutes with no account required, useful for AI agent workflows that need ephemeral live endpoints for testing — Simon Willison validated this using GPT-5.5 xhigh in Codex Desktop.

- [Good results fine tuning a local LLM like Qwen 3:0.6B to categorize questions](https://www.teachmecoolstuff.com/viewarticle/fine-tuning-a-local-llm-to-categorize-questions) — Concrete case study: a 0.6B model went from ~10% to reliable accuracy on classification after fine-tuning with ~850 examples using Unsloth/QLora. Practical evidence that tiny local models can be specialized for pre-processing steps in AI pipelines (e.g., routing, intent classification) rather than relying on larger, costlier models for every task.

- [Local LLM Inference Optimization: The Complete Guide](https://www.reddit.com/r/LocalLLaMA/comments/1uc3wg9/local_llm_inference_optimization_the_complete/) — A year of llama.cpp experiments distilled into a practical guide covering VRAM fitting, KV cache, MoE layer placement, MTP, and OOM traps. Worth feeding directly to your AI assistant when configuring local model infrastructure.

## Try This

Load the [codebase-memory-mcp](https://github.com/DeusData/codebase-memory-mcp) server into Claude Code on your current project: it indexes your entire repo into a persistent knowledge graph in milliseconds, which means Claude Code will answer architecture and cross-file questions with dramatically fewer tokens and fewer hallucinated file paths. Configure it once, point it at your repo root, and then open a Claude Code session and ask "what are the ten most-called functions across this codebase and which modules depend on them?" — you'll see the difference in answer quality versus a cold session immediately.
