## Engineering & AI Workflows
- [Constraint Decay: The Fragility of LLM Agents in Back End Code Generation](https://arxiv.org/abs/2605.06445) — Research finding that LLM agents progressively forget or violate constraints they were given earlier in long agentic coding sessions — directly relevant if you're using Claude Code or Codex for multi-step backend tasks. Expect constraint drift on long runs and build explicit re-anchoring steps into your prompts.
- [Quoting Armin Ronacher](https://simonwillison.net/2026/May/24/armin-ronacher/#atom-everything) — Flask/Rye creator documents a new problem: AI-filed GitHub issues that contain confident but wrong root cause analysis, fake minimal repros, and hallucinated error classes. If you're using agents to file issues or bug reports, strip AI prose and enforce the format: observed behaviour, expected behaviour, exact error output.
- [server: fix checkpoints creation by jacekpoplawski](https://www.reddit.com/r/LocalLLaMA/comments/1tn0jyp/server_fix_checkpoints_creation_by_jacekpoplawski/) — Detailed write-up on how agentic coding tools that rewrite conversation history (e.g. for context optimisation) can force full prompt reprocessing of 70k+ token contexts in llama.cpp. The fix: prefer tools that don't mutate history, and enable "preserve thinking" for reasoning models like Qwen 3.6.
- [🚀 Skills for small businesses, officially released by Anthropic](https://www.reddit.com/r/ClaudeAI/comments/1tm94ai/skills_for_small_businesses_officially_released/) — Anthropic's 31 small-business skill packs (workflow, memory, connectors, operating rules as `.md` files) hit 382k downloads on day one. The underlying skill file format works with other agents too — worth studying the structure for your own agent system prompts.

## Try This
Drop the [multica-ai/andrej-karpathy-skills](https://github.com/multica-ai/andrej-karpathy-skills) CLAUDE.md into the root of your next project before starting a Claude Code session. Review it first — treat it as a prompt template you're steering, not a black box — and add one or two project-specific constraints at the top (e.g. "never mutate existing tests", "all new functions must have explicit return types"). This directly addresses the constraint decay problem flagged in today's research: front-loading behavioural rules in the project-level CLAUDE.md means they're in context from the first token rather than fading as the session grows.
