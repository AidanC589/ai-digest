## Engineering & AI Workflows
- [AIEWF Daily Dispatch: Autoresearch and the tension between AI and human agency](https://www.latent.space/p/aiewf-daily-dispatch-agency) — The clearest articulation yet of the inner/outer loop split: agents own execution (inner loop), engineers own direction and understanding (outer loop). Notion's Geoffrey Litt argued that delegating *understanding* — not just execution — is what gets engineers replaced; those who understand what agents produce will keep having the next big idea.

- [Autoresearch: The feedback loop behind self-improving agents](https://www.latent.space/p/autoresearch-introspection) — Introspection's concept of "agent recipes" — portable bundles of evals, judges, signal processing, and domain knowledge — is a concrete pattern for encoding your own expertise into a loop that can improve without constant human bottlenecking. Worth thinking about what your equivalent of a recipe looks like for your review process.

- [How Cursor deploys AI inside the enterprise](https://www.latent.space/p/cursor-forward-deployed-engineers) — Cursor's FDE team (targeting 10x growth by December) works on-site to build full software factory pipelines covering plan → write → review → test → deploy. The role requires engineers with 5+ years of production experience — a signal that AI-workflow expertise alone isn't enough; domain credibility matters when selling the vision internally.

- [Warp CEO Zach Lloyd on why software factories are the next phase of coding](https://www.latent.space/p/software-factories) — Warp's new Oz platform targets full automation of the triage → implement → review → verify → ship loop across local and cloud sandboxes. Lloyd expects most significant software projects to run some form of automated factory within a year — the practical implication is that wiring your current agent workflows into a continuous loop, not just one-shot tasks, is the near-term priority.

## Tool Updates
- [Claude Code v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198) — Large release. Key changes for AI-assisted dev workflows: background agents now automatically commit, push, and open a draft PR when they finish work in a worktree (no more manual handoff); subagents and context compaction now inherit extended thinking config, improving quality on delegated tasks; the Explore agent now inherits the main session's model (capped at Opus) instead of defaulting to Haiku; transient network errors (ECONNRESET etc.) now retry with backoff instead of aborting the turn; new `/dataviz` skill added; Claude in Chrome is now GA. Several agent reliability fixes for stuck sessions and dead API error states.

- [Codex CLI 0.143.0-alpha.33](https://github.com/openai/codex/releases/tag/rust-v0.143.0-alpha.33) — Alpha release with no changelog detail published.

## Try This
Claude Code v2.1.198 now lets subagents inherit your session's extended thinking configuration — which previously they didn't. Take a complex task you'd normally run as a single long session and break it into a lead agent plus two or three subagents (e.g. one for implementation, one for writing tests, one for the PR description), then start the session with extended thinking enabled. The subagents will now reason at the same depth as your main session rather than defaulting to faster, shallower output, giving you meaningfully better quality on delegated work without any extra prompt engineering.
