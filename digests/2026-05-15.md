## Tool Updates

- [Claude Code v2.1.142](https://github.com/anthropics/claude-code/releases/tag/v2.1.142) — Fast mode now defaults to Opus 4.7 (set `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1` to pin to 4.6). New `claude agents` flags include `--add-dir`, `--settings`, `--mcp-config`, `--model`, `--effort`, and `--permission-mode` for configuring background sessions. Critical bug fixes: background sessions now survive macOS sleep/wake (daemon detects clock jumps), fixed crash-loop on binary upgrades (e.g. `brew upgrade`), fixed Windows deadlock on network drives, and fixed `MCP_TOOL_TIMEOUT` not actually raising the fetch timeout beyond 60 seconds. Also improved reactive compaction to avoid wasted near-full-context retries.

- [Codex rust-v0.131.0-alpha.16 through alpha.19](https://github.com/openai/codex/releases/tag/rust-v0.131.0-alpha.19) — Four alpha releases in quick succession; no detailed changelogs published, indicating active but pre-stable iteration on the Rust rewrite.

## Try This

This week, test Claude Code's new `--effort` flag on a background agent task you'd normally run interactively. Spin up a session with `claude --bg --effort high` for a complex refactor or spec-to-code task, then use the mobile Codex app (or just check back in) to review and steer the output remotely. The combination of the macOS sleep/wake fix in v2.1.142 and the new Opus 4.7 Fast mode default means background sessions are now meaningfully more reliable for long-running work — this is a good week to actually trust one with something non-trivial.
