# AI Digest — 2026-06-07

## TL;DR
Meta's AI chatbot had a severe security flaw that let attackers hijack 20,000+ Instagram accounts via manipulated password resets — a direct lesson in what happens when AI tools skip input validation. Claude Code shipped another bug-fix release this week.

## General AI Developments
- [Meta confirms 1000s of Instagram accounts were hacked by abusing its AI chatbot](https://this.weekinsecurity.com/meta-confirms-thousands-of-instagram-accounts-were-hacked-by-abusing-its-ai-chatbot/) — Meta's AI-assisted account recovery system failed to verify that the email address requesting a password reset matched the one on file; the chatbot simply complied when given a different address. Over 20,000 accounts were compromised across roughly two months. A sharp reminder that AI tools integrated into auth flows need explicit, hardened validation logic that the AI itself cannot bypass via conversation.

- [Cohere's unreleased coding model (early access for localllama)](https://www.reddit.com/r/LocalLLaMA/comments/1tylzy2/coheres_unreleased_coding_model_early_access_for/) — Cohere is community-testing a 30B coding model (3B active params via MoE) before official launch; weights are live on HuggingFace. Worth watching as a potential local coding assistant alternative, though no benchmark comparisons are published yet.

## Engineering & AI Workflows
- [POV: Every time you typo and Claude overthinks it](https://www.reddit.com/r/ClaudeAI/comments/1tz0djm/pov_every_time_you_typo_and_claude_overthinks_it/) — A pattern worth knowing: Claude's extended thinking mode can spiral on ambiguous or incomplete prompts, burning tokens on internal contradictions rather than asking a clarifying question. System prompt instructions to "ask when requirements seem incomplete" are being ignored roughly half the time in practice. Workaround: explicitly state "if anything is unclear or contradictory, stop and ask before proceeding" as the first line of your system prompt.

- [Meta confirms 1000s of Instagram accounts were hacked by abusing its AI chatbot](https://this.weekinsecurity.com/meta-confirms-thousands-of-instagram-accounts-were-hacked-by-abusing-its-ai-chatbot/) — The technical failure here — an AI tool that bypassed a separate validation code path — is a direct analogue to AI-generated code that looks correct but skips an auth check in a non-obvious branch. When reviewing AI-generated code in security-sensitive flows, explicitly prompt for "what input validation assumptions does this code make, and what happens if they are violated?"

## Try This
When prompting Claude (or any extended-thinking model) for code in security-sensitive flows, add this line at the top of your prompt: *"Before writing any code, list every input validation assumption this implementation makes and explicitly note what an attacker could provide to violate each one."* This forces the model to surface its own blind spots before generating the code rather than after, and gives you a checklist to verify during review — the exact step that was missing in Meta's AI auth flow.
