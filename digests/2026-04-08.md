## Try This

Audit your standing prompt templates for framing polarity. Based on the LLM question-framing research: pick one recurring prompt you use for code review, security checks, or architectural decisions and deliberately rewrite it in the opposite framing (e.g., "What could go wrong with this approach?" vs. "What's correct about this approach?") then run both against the same code. If you get materially different conclusions from identical context, that's a systematic gap in your review process — and it's a signal to build a two-pass "steelman + critic" prompt structure into your workflow rather than relying on a single framing.
