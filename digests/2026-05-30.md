## Engineering & AI Workflows

- [[AINews] Founders and Forward Deployed Engineers](https://www.latent.space/p/ainews-founders-and-forward-deployed) — Critical finding for anyone building multi-turn RL agent loops: a widely-shared HuggingFace deep-dive confirmed many tool-using training pipelines are silently broken because re-tokenizing parsed tool calls changes token sequences, meaning gradients are applied to sequences the model never actually sampled. The fix is strict "Token-In, Token-Out": never re-encode sampled tokens. Also: Anthropic shipped mid-conversation system instructions that preserve prompt cache — directly useful for long-running Claude Code sessions and cost control.

- [[AINews] Founders and Forward Deployed Engineers](https://www.latent.space/p/ainews-founders-and-forward-deployed) — *(synthesised above — see also:)* Opus 4.8 real-world verdict from practitioners: less over-agentic and more cooperative than 4.7 in coding tasks (per Jeremy Howard), but CursorBench shows it marginally worse than 4.7 within margin of error, and there are documented regressions on content faithfulness in document parsing. Treat it as a quality-of-life release, not a capability step-change.

- [Various LLM Smells](https://shvbsle.in/various-llm-smells/) — Community-sourced catalogue of failure patterns in LLM output (356 upvotes, 283 comments on HN) — useful reference for building review checklists or prompt guards when steering AI-generated code.

- [Fed up with vibe coders, dev sneaks data-nuking prompt injection into their code](https://www.reddit.com/r/LocalLLaMA/comments/1trdnap/fed_up_with_vibe_coders_dev_sneaks_datanuking/) — A developer embedded a malicious prompt injection into open-source code specifically targeting AI coding agents. Direct security threat to AI-assisted workflows that pull in third-party packages without human review of every file — reinforces the need for sandboxed execution and supply-chain scrutiny when Claude Code operates autonomously.

- [CAPTCHAs can still detect AI agents](https://research.roundtable.ai/captchas-detect-ai/) — Research shows frontier models (Claude, GPT, Gemini) are *less* humanlike in their process signatures than smaller models like Qwen — larger-scale RLHF/output fine-tuning diverges from human cognitive patterns. Practical implication: AI agents operating in web-scraping or browser-automation workflows remain detectable regardless of output quality.

## Tool Updates

- [Claude Code v2.1.157](https://github.com/anthropics/claude-code/releases/tag/v2.1.157) — Large release: plugins in `.claude/skills` now auto-load without a marketplace; `claude plugin init <name>` scaffolds new plugins locally; the `agent` field in `settings.json` is now honored for dispatched sessions with `--agent <name>` override; numerous bug fixes including corrupt image crashes, Esc not cancelling agent open, orphaned worktrees after retention sweeps, and clipboard issues in tmux. Also adds `OTEL_LOG_TOOL_DETAILS=1` for tool parameter telemetry.

- [Claude Code v2.1.158](https://github.com/anthropics/claude-code/releases/tag/v2.1.158) — Auto mode (previously only on direct Anthropic API) is now available on Bedrock, Vertex, and Foundry for Opus 4.7 and 4.8. Opt in with `CLAUDE_CODE_ENABLE_AUTO_MODE=1`.

## Try This

With Claude Code v2.1.157's new auto-loading skills directory, create a `prompt-injection-scanner.md` skill in `.claude/skills/` that instructs Claude to automatically flag any third-party file it reads during a task if it contains instruction-like language directed at an AI agent (phrases like "ignore previous instructions," "you are now," or "disregard your system prompt"). This turns every `cat` or file-read operation into a passive security scan — no extra prompting required per session. Drop the skill file in your repo and it loads automatically from now on.
